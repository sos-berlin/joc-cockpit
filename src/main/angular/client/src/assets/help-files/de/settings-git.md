# Einstellungen - Git

Das JOC Cockpit bietet [JS7 - Inventory Git Integration](https://kb.sos-berlin.com/display/JS7/JS7+-+Inventory+Git+Integration) für die Versionierung und das Rollout von Zeitplan-Objekten unter Verwendung von Git-Repositories.

Die Seite *Einstellungen* ist über das Symbol ![wheel icon](assets/images/wheel.png) in der Menüleiste zugänglich.

Die folgenden Einstellungen werden für die Verwendung mit Git-Repositories übernommen. Änderungen werden sofort wirksam.

## Git-Repository-Einstellungen

### Einstellung: *git\_hold\_workflows*, Standard: *rollout*

Gibt an, ob Arbeitsabläufe als lokal in einer Umgebung oder für das Rollout mit Git vorgesehen sind.

### Einstellung: *git\_hold\_resource\_locks*, Standard: *rollout*

Gibt an, ob [Resource Locks](/resources-resource-locks) als lokal für eine Umgebung oder für den Rollout mit Git vorgesehen ist.

### Einstellung: *git\_hold\_file\_order\_sources*, Voreinstellung: *rollout*

Legt fest, ob Quellen für Dateiaufträge als lokal in einer Umgebung oder für den Rollout mit Git vorgesehen sind.

### Einstellung: *git\_hold\_notice\_boards*, Standard: *rollout*

Gibt an, ob [Notice Boards](/resources-notice-boards) als lokal für eine Umgebung oder für das Rollout mit Git vorgesehen ist.

### Einstellung: *git\_hold\_script\_includes*, Voreinstellung: *rollout*

Gibt an, ob Script Includes als lokal für eine Umgebung oder für das Rollout mit Git vorgesehen sind.

### Einstellung: *git\_hold\_job\_templates*, Voreinstellung: *rollout*

Gibt an, ob Job Templates als lokal in einer Umgebung oder für das Rollout mit Git vorgesehen sind.

### Einstellung: *git\_hold\_job\_resources*, Voreinstellung: *lokal*

Legt fest, ob Job-Ressourcen als lokal in einer Umgebung oder für den Rollout mit Git bestimmt sind.

### Einstellung: *git\_hold\_calendars*, Voreinstellung: *lokal*

Legt fest, ob Kalender als lokal in einer Umgebung betrachtet werden oder ob sie für den Rollout mit Git vorgesehen sind.

### Einstellung: *git\_hold\_schedules*, Voreinstellung: *lokal*

Legt fest, ob Zeitpläne als lokal in einer Umgebung betrachtet werden oder für das Rollout mit Git vorgesehen sind.
