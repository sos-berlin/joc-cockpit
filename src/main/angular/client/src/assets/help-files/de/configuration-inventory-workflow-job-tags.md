# Konfiguration - Inventarisierung - Arbeitsabläufe - Job-Kennzeichnungen

Im Bereich *Arbeitsablauf* können Sie Arbeitsabläufe aus einer Folge von Anweisungen erstellen. Sie können die *Job Anweisung* per Drag&amp;Drop aus der *Funktionsleiste* an eine Position im Arbeitsablauf ziehen.

Die grafische Benutzeroberfläche bietet eine Reihe von Registerkarten für die Angabe von Details. Die dritte Registerkarte wird für *Job-Kennzeichnungen* angeboten.

## Job-Kennzeichnungen

Einem Job kann eine beliebige Anzahl von Kennzeichnungen hinzugefügt werden. Sie werden in der Ansicht [Arbeitsabläufe](/workflows) angezeigt und sind optional in den Benachrichtigungen enthalten. Einzelheiten finden Sie unter [JS7 - Tagging Jobs](https://kb.sos-berlin.com/display/JS7/JS7+-+Tagging+-+Jobs).

- **Kennzeichnungen** können hinzugefügt oder gelöscht werden. Sie können aus der Liste der vorhandenen Kennzeichnungen ausgewählt werden. Wenn Sie einen neuen Namen eingeben, wird die Kennzeichnung erstellt.

## Kennzeichnungsgruppen

Kennzeichnungsgruppen können verwendet werden, um Kennzeichnungen zu organisieren, die einer gemeinsamen Gruppe angehören sollen. Einem Job soll beispielsweise eine der Ticketprioritäten P1, P2, P3 oder P4 zugewiesen werden, je nachdem, wie kritisch der Job ist. Benutzer können die Kennzeichnungsgruppe und die Kennzeichnung durch einen Doppelpunkt getrennt eingeben, z.B. *TicketPriorität:P1*, *TicketPriorität:P2* usw. Die Ticketpriorität eines Jobs wird verfügbar, wenn im Falle eines Fehlers Benachrichtigungen erstellt werden, und kann zur Versorgung eines Ticketsystems verwendet werden.

- **Kennzeichnungsgruppe:Kennzeichnung** ermöglicht das Hinzufügen oder Löschen der entsprechenden Kennzeichnung in der angegebenen Kennzeichnungsgruppe. Die Kennzeichnungsgruppe wird erstellt, wenn sie nicht existiert.
