# Konfiguration - Inventarisierung - Workflow - Job Tags

Im Bereich *Workflow* können Sie Workflows aus einer Folge von Anweisungen erstellen. Sie können die *Auftragsanweisung* per Drag &amp; Drop aus der *Symbolleiste* an eine Position im Workflow ziehen.

Die grafische Benutzeroberfläche bietet eine Reihe von Registerkarten für die Angabe von Auftragsdetails. Die dritte Registerkarte wird für *Job Tags* angeboten.

*Tags

Einem Auftrag kann eine beliebige Anzahl von Tags hinzugefügt werden. Sie werden in der Ansicht [Workflows](/workflows) angezeigt und sind optional in den Benachrichtigungen enthalten. Einzelheiten finden Sie unter [JS7 - Tagging Jobs](https://kb.sos-berlin.com/display/JS7/JS7+-+Tagging+-+Jobs).

- **Tags** ermöglicht das Hinzufügen oder Löschen von Tags. Sie können aus der Liste der vorhandenen Tags ausgewählt werden. Wenn Sie einen Tag-Namen eingeben, wird der Tag erstellt.

## Tag-Gruppen

Tag-Gruppen können verwendet werden, um Tags zu organisieren, die einer gemeinsamen Gruppe angehören sollen. Einem Job sollte beispielsweise eine der Ticketprioritäten P1, P2, P3 oder P4 zugewiesen werden, je nachdem, wie kritisch der Job ist. Benutzer können die Tag-Gruppe und den Tag durch einen Doppelpunkt getrennt eingeben, z.B. *TicketPriorität:P1*, *TicketPriorität:P2* usw. Die Ticketpriorität eines Auftrags wird verfügbar, wenn im Falle eines Fehlers Benachrichtigungen erstellt werden, und kann zur Fütterung eines Ticketsystems verwendet werden.

- **Tag-Group:Tag** ermöglicht das Hinzufügen oder Löschen des entsprechenden Tags in der angegebenen Tag Group. Die Tag-Gruppe wird erstellt, wenn sie nicht existiert.
