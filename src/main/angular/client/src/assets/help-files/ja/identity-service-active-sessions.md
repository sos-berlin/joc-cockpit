# IDサービス管理 - アクティブセッション

IDサービスは、認証と認可によってJOCコックピットへのアクセスを管理します。[IDサービス管理](/identity-services) を参照してください。

ユーザーは、メニューバーのホイールアイコンから *アカウント管理 - アクティブセッション* 画面で、アクティブなセッションを保持しているアカウントを識別できます。

アクティブなセッションは、使用中のアカウント、ログインに使用した IDサービス、および残りのセッション時間が表示されます。

- JOCコックピットはアカウントごとのセッション数を制限しません。
- セッションの残り時間は、以下によって制限されます：
  - [設定 - Identity Service](/settings-identity-service) 画面の *idle_session_timeout* は、ユーザーアクティビティなしでセッションが有効な状態を維持できる最大時間を制限します。
  - OIDCやKeycloakなどのIDサービスプロバイダーは、ユーザセッションの最大継続時間を制限できます。

## アクティブなセッションに対する操作

アクティブなセッションに対する操作は以下のとおりです：

- **ブロックリストに追加**は、関連するアカウントを今後のログインを拒否する[IDサービス管理 - ブロックリスト](/identity-service-blocklist) に追加します。この操作はアカウントの現在のセッションを終了させません。
- **セッションキャンセル**は、アカウントの現在のセッションを強制的に終了します。これは、アカウントが新しいログイン操作を実行することを妨げません。
- **アカウントの全セッションをキャンセル** **セッションキャンセル**と同様に、指定されたアカウントのすべてのセッションを終了します。

1つ以上のセッションを選択している場合、**セッションキャンセル** 操作は、画面右上隅にあるボタンで一括操作から利用できます。
