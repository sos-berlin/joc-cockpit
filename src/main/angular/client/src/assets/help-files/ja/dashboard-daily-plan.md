# 実行計画

デイリープラン] パネルには、実行計画によって作成されたオーダーの約定情報が表示されます。ユーザーの操作によりオンデマンドで作成されたオーダーや、*File Order Sources*によりファイル ウォッチングから作成されたオーダーは除外されます。

<img src="dashboard-daily-plan.png" alt="Daily Plan" width="330" height="80" />

## 実行計画ステータス

実行計画ステータスは、オーダーがデイリープランサービスによって作成されたときの初期ステータスです。

- **予定**オーダーはコントローラーおよびエージェントに送信されていません。予定] オーダーの数は、日付範囲がオーダーを送信すべき日数の範囲内にある場合に問題があることを示します。
- **提出済み** オーダーは、その日のうちに実行される予定であるか、実行中です。このステータスは、開始されたオーダーの現在の状態を反映するものではなく、日中に実行されるはずのオーダーを要約したものです。
- **完了** オーダーは完了しました。これは、**Order History*ビューで表示される、オーダーが正常に完了したか失敗したかの結果とは無関係です。

表示されているオーダー数をクリックすると、*実行計画*ビューに移動し、オーダーの詳細が表示されます。

## フィルター

パネルの右上にあるドロップダウンボタンは、日付範囲からオーダーを選択することができます：

- **今日** オーダーは、ユーザープロフィールのタイムゾーンから計算される現在の日に関連します。
- **翌日** オーダーは、翌日の約定を対象としています。ただし、**本日の** オーダーは除きます。
- **翌々日のオーダーは、翌々日の約定を対象とします。
- **翌3日目**のオーダーは、翌3日目の約定を対象とします。
- **翌4日目**のオーダーは、翌4日目の約定を対象とします。
- **翌5日目のオーダーは翌5日目の約定となります。
- **翌6日目のオーダーは、翌6日目の約定が対象となります。
- **翌7日目のオーダーを対象とします。
