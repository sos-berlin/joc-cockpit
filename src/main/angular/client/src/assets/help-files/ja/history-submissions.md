# 投稿履歴

提出履歴*は、[Daily Plan](/daily-plan) から提出されたオーダーを記録します。

オーダーは実行計画によって2つのステップで作成されます: まず*計画*され、次にコントローラーとエージェントに*提出*されます。サブミッションには、オーダーがエージェントによって自律的に開始されることも含まれます。

提出履歴*は、[Cleanup Service](/service-cleanup) によって実行されるデータベースのパージの対象となります。

## 履歴パネル

実行計画日ごと、サブミッションごと、オーダーごとにブロック分けされて表示されます。

[Profile - Preferences](/profile-preferences) から特に指定がない場合、最大5000件まで表示されます。

### 実行計画の履歴

実行計画日ごとに以下の情報が表示されます。

- **実行計画日**は、オーダーがスケジュールされた日を示します。
- **総カウント数**は、指定された日付のすべての送信対象となったオーダーの数を示します。
- **送信回数**は、正常に送信されたオーダーの数を示します。
  - 合計件数**と一致する場合、すべてのオーダーが正常に送信されています。
  - 件数がゼロ以上で*総件数*以下の場合は、注文を個別に送信する設定になっています。
    - オーダーを個別に送信する設定になっている場合、[Settings - Daily Plan](/settings-daily-plan) を参照してください。
    - を参照してください。
  - 件数がゼロの場合は
    - オーダーを個別に送信する設定になっていない場合、[Settings - Daily Plan](/settings-daily-plan) を参照。
    - すべてのオーダーを送信できませんでした。

### 提出履歴

実行計画日には、いくつでも提出が可能です。ユーザーがワークフローやスケジュールなどのインベントリオブジェクトに変更を展開し、実行計画を更新するオプションを選択すると、指定されたオブジェクトに提出が追加されます。

実行計画日*の矢印をクリックすると、実行計画ごとの詳細が表示されます：

- **総サブミッションカウント(Total Submission Count)**は、指定されたサブミッションの対象となるオーダーの数を示します。
- **サブミッションカウント**は、指定されたサブミッションの範囲内で正常に送信されたオーダーの数を示します。
  - 件数が*Total Submission Count*と一致する場合、すべてのオーダーが正常に送信されています。
  - 件数がゼロまたはゼロ以上の場合は、実行計画日ごとの提出件数に関するこれまでの説明が適用されます。

### オーダーごとの提出履歴

提出日*の矢印をクリックすると、オーダーごとの詳細が表示されます：

- **メッセージ**は、提出に失敗した場合のエラーメッセージです。
- **オーダーID**は、オーダーに割り当てられた一意の識別子です。
- **ワークフロー**はオーダーが通過したワークフローを示します。
  - ワークフロー名をクリックすると、[Workflows](/workflows) ビューに移動します。
  - 鉛筆アイコンをクリックすると、[Configuration - Inventory - Workflows](/configuration-inventory-workflows) のビューに移動します。
- **スケジュール]は、オーダーが開始される日時を示します。
- **ステータス**は、*提出済み*または*未提出*のいずれかです。
  - *Submitted]は、オーダーがエージェントで利用可能であることを示します。
  - *未提出]は送信に失敗したことを示します。

## フィルタ

ユーザーはウィンドウの上部にあるフィルターを適用して、実行計画の日付と提出の表示を制限することができます。

- **提出済み**、未提出**のフィルターボタンは、関連するステータスの提出物に表示を制限します。
- **日付範囲**フィルターボタンは、提出物の表示範囲を選択できます。
- **現在のコントローラー**チェックボックスは、現在選択されているコントローラーに提出を制限します。
