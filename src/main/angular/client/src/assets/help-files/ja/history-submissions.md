# 履歴 - 実行計画

*履歴 - 実行計画* は、[実行計画](/daily-plan) から発行されたオーダーを表示します。

オーダーは実行計画によって2つのステップで作成されます: まず *計画* され、次にコントローラーとエージェントに *登録* されます。*登録* には、オーダーがエージェントによって自律的に開始されることも含まれます。

*履歴 - 実行計画* は、[クリーンアップサービス](/service-cleanup) によって実行されるデータベースの削除の対象となります。

## 履歴パネル

実行計画日ごと、登録ごと、オーダーごとにブロック分けされて表示されます。

[プロファイル - 設定値](/profile-preferences) から特に指定がない場合、最大5000件まで表示されます。

### 実行計画の履歴

実行計画日ごとに以下の情報が表示されます。

- **実行計画日付** は、オーダーが計画された日を示します。
- **合計数** は、指定された日付のすべての登録対象となったオーダーの数を示します。
- **登録数** は、正常に登録されたオーダーの数を示します。
  - *合計数* と一致する場合、すべてのオーダーが正常に登録されています。
  - 件数がゼロ以上で *合計数* 以下の場合は、オーダーを個別に登録する設定になっています。
    - オーダーを個別に登録する設定になっている場合、[設定 - Daily Plan](/settings-daily-plan) を参照してください。
    - 一部のオーダーを登録できていません。
  - 件数がゼロの場合は
    - オーダーを個別に登録する設定になっていない場合、[設定 - Daily Plan](/settings-daily-plan) を参照してください。
    - すべてのオーダーを登録できていません。

### 登録履歴

実行計画実行日には、オーダーはいくつでも登録が可能です。ユーザーがワークフローやスケジュールなどのインベントリーオブジェクトに変更を展開し、実行計画を更新するオプションを選択すると、指定されたオブジェクトに追加登録されます。

*実行計画日付* の三角アイコンをクリックすると、実行計画ごとの詳細が表示されます：

- **合計数** は、指定された登録の対象となるオーダーの数を示します。
- **登録数** は、指定された登録の範囲内で正常に登録されたオーダーの数を示します。
  - 件数が *合計数* と一致する場合、すべてのオーダーが正常に登録されています。
  - 件数がゼロまたはゼロ以上の場合は、実行計画日ごとの登録件数に関するこれまでの説明が適用されます。

### オーダーごとの登録履歴

*登録日* の三角アイコンをクリックすると、オーダーごとの詳細が表示されます：

- **メッセージ** は、登録に失敗した場合のエラーメッセージです。
- **オーダーID** は、オーダーに割り当てられた一意の識別子です。
- **ワークフロー** はオーダーが実行したワークフロー名を示します。
  - ワークフロー名をクリックすると、[ワークフロー](/workflows) 画面に移動します。
  - 鉛筆アイコンをクリックすると、[設定 - インベントリー - ワークフロー](/configuration-inventory-workflows) 画面に移動します。
- **スケジュール済** は、オーダーが開始される日時を示します。
- **ステータス** は、*登録* または *未登録* のいずれかです。
  - *登録* は、オーダーがエージェントで利用可能であることを示します。
  - *未登録* は、登録に失敗したことを示します。

## 詳細検索

ユーザーはウィンドウの上部にある詳細検索ボタンから、実行計画の日付と登録の表示を制限することができます。

- **登録済**、**未登録** のチェックボックスは、そのステータスに表示を制限します。
- **期間** のチェックボックスは、表示範囲を選択できます。
- **コントローラーID** ボックスは、現在選択されているコントローラーに制限します。
