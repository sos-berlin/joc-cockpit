# タスク履歴

*タスク履歴* から2-3秒毎に更新される実行ログを表示できます。これにより、ジョブのログ出力をほぼリアルタイムで追うことができます。

## ログ表示設定

ログ表示では、ウィンドウの上部から様々な条件によるフィルタリングが可能です：

- **Main** はジョブの開始に関する詳細とジョブの開始時のパラメーターを表示します。このような出力は [MAIN] という修飾子で示されます。
- **stdout** はジョブが標準出力チャンネルに書き込んだ出力が [STDOUT] を使って表示されるかどうかを指定します。
- **debug** は [JS7 - Job API](https://kb.sos-berlin.com/display/JS7/JS7+-+Job+API)を使用するJavaジョブに、*debug* または *trace* 値を持つ *log_level* 引数を追加することができます。デバッグ出力がジョブから利用可能な場合、[DEBUG]修飾子を使用して表示されます。

ジョブが標準エラーチャンネルに書き込んだログ出力はフィルタリングの対象にはならず、[STDERR] 修飾子を使って表示されます。

## ログ出力の表示

ログ出力は到着順に表示されます。

### タイムスタンプ

ログ出力には、さまざまなソースからのタイムスタンプが表示されます：

- **エージェント時刻**：開始]や[終了]などのイベントはエージェントによって作成され、エージェントのリアルタイムクロックを反映します。
- **ジョブ時刻**：ジョブの出力は、ジョブが実行されるサーバのタイムゾーン、またはジョブの実装から指定されたタイムゾーンを使用します。

*タスクログビュー* はタイムスタンプをユーザのタイムゾーンに変換します。 [プロファイル - 設定値](/profile-preferences) の*ログタイムスタンプにタイムゾーンを使用* 設定が有効な場合、エージェントのタイムゾーンが使用されます。
エージェントのリアルタイムクロックが同期されていない場合、ログ出力のタイムスタンプが不正確になる可能性があります。
