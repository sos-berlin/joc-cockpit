# 実行計画 - カレンダーパネル

実行計画カレンダーパネルから様々な操作が可能です。 

実行計画から利用できる一般的な操作については、[実行計画](/daily-plan) をご覧ください。

## 日付の選択

カレンダーの日付をクリックすると、選択した日付のオーダーが表示されます。

## 複数日付の選択

複数の日付を選択するには

- マウスを押したままドラッグして日付範囲を選択します、
- またはCtrlキーを押しながらマウスをクリックし、開始日と終了日を選択します、
- またはカレンダーアイコンをクリックし、マウスクリックで開始日と終了日を選択します。

選択された日付はハイライトされ、メインメニューの下に *オーダー削除* と *オーダーキャンセル* ボタンが表示されます。

以下のボタンは、操作の範囲を制限します： 

- **全て**：どのステータスのオーダーにも適用できます。
- **予定**：コントローラーに *登録* されていない *予定* のオーダーに *登録* と *削除* の 操作を適用できます。
- **登録**：コントローラーおよびエージェントに *登録* されたオーダーに適用できます。
- **終了**：キャンセル操作は、完了したオーダーに適用できます。
- **遅延**は、オーダーが予定より遅れて開始されたことを表示します。

### オーダーキャンセル

- 選択した日付範囲内の *登録* オーダーに適用すると、コントローラーとエージェントからオーダーが呼び戻されます。
- *登録* または *完了* のオーダーに適用すると、そのオーダーは *予定* のステータスになります。
- *予定* のオーダーは無視されます。

### オーダー削除

- *予定* オーダーに適用すると、オーダーは実行計画から削除されます。
  - オーダーが実行計画の日付から削除されると、そのオーダーは実行されず、デイリープランサービスは指定された日付にオーダーを追加しようとしません。
  - *登録削除* は暗黙的にオーダーを削除します。さらに、指定された実行計画の日付のオーダーはすべて削除され、デイリープランサービスの次回実行時に指定された日付のオーダーが計画されます。
- この操作は、*登録* および *終了* のオーダーに対しては無視されます。

### 実行計画作成

この操作は、カレンダーパネルの下にあるボタンから、個々の日付および日付範囲に対して実行できます。

- 選択した日に実行計画が作成されます。
  - すべてのオーダーを作成するか、選択したスケジュールとワークフローからオーダーを作成するかを選択できます。
  - 同じスケジュールから既存のオーダーを置き換えたり、コントローラーに即座にオーダーを送信したりすることができます。
  - ユーザーは、デイリープランサービスで考慮するように設定されていないスケジュールからのオーダーを含めることができます。
- 指定された日付の実行計画が作成された場合、デイリープランサービスの次の実行では、同じ日付の追加オーダーは計画されません。[設定 - Daily Plan](/settings-daily-plan) ページを参照してください。

### 登録削除

この操作は、カレンダーパネルの下にあるボタンから、個々の日付および日付範囲に対して実行できます。

- 選択した日の実行計画は、*登録* または *終了* オーダーがない場合に削除されます。*予定* のオーダーがある場合は、実行計画と一緒に削除されます。
- 指定した日の実行計画が削除された場合、デイリープランサービスの次の実行では、その日のオーダーが計画されます。[設定 - Daily Plan](/settings-daily-plan) を参照してください。
