# 実行計画カレンダー

実行計画カレンダーから様々な操作が可能です。 

実行計画から利用できる一般的な操作については、[Daily Plan](/daily-plan) をご覧ください。

## 単一の日付の選択

カレンダーの日付をクリックすると、選択した日付のオーダーが表示されます。

## 複数の日付の選択

複数の日付を選択するには

- マウスを押したままドラッグして日付範囲を選択します、
- またはCtrlキーを押しながらマウスをクリックし、開始日と終了日を選択します、
- またはカレンダーアイコンをクリックし、マウスクリックで開始日と終了日を選択します。

選択された日付はハイライトされ、メインメニューの下に *Remove Order* と *Cancel Order* ボタンが表示されます。

以下のフィルターボタンは、操作の範囲を制限します： 

- **すべて**：すべて**: どのステータスのオーダーにも適用できます。
- **計画中**：予定**: コントローラに*送信*されていない*予定*のオーダーに*送信*と*削除*の 操作を適用できます。
- **提出済み**：キャンセル**操作は、コントローラおよびエージェントに*送信*されたオーダーに適用できます。
- **完了**：完了**: **キャンセル**操作は、完了したオーダーに適用できます。
- **Late**は、オーダーが予定より遅れて開始されたことを示す、オーダー状態の追加フィルタです。

### オーダーのキャンセル

- 選択した日付範囲内の*送信済み*オーダーに適用すると、コントローラーとエージェントからオー ダーが呼び戻されます。
- 提出済み] または [完了] のオーダーに適用すると、そのオーダーは [計画中] のステータスになります。
- 計画中*のオーダーは無視されます。

### オーダーの削除

- 予定*オーダーに適用すると、オーダーは実行計画から削除されます。
  - オーダーが日計 画の日付から削除されると、そのオーダーは実行されず、日計 画サービスは指定された日付にオーダーを追加しようとしません。
  - 日計 画の削除] 操作は暗黙的にオーダーを削除します。さらに、指定された日次計画の日付の注文はすべて削除され、日次計画サービスの次回実行時に指定された日付のオーダーが計画されます。
- この操作は、*送信済み*および*完了*のオーダーに対しては無視されます。

### 日次計画の作成

この操作は、カレンダーウィジェットの下にあるボタンから、個々の日付および日付範囲に対して実行できます。

- 選択した日に実行計画が作成されます。
  - すべてのオーダーを作成するか、選択したスケジュールとワークフローからオーダーを作成するかを選択できます。
  - 同じスケジュールから既存のオーダーを置き換えたり、コントローラーに即座にオーダーを送信したりすることができます。
  - ユーザーは、実行計画サービスで考慮するように設定されていないスケジュールからのオーダーを含めることができます。
- 指定された日付の実行計画が作成された場合、日次計画サービスの次の実行では、同じ日付の追加オーダーは計画されません。[Settings - Daily Plan](/settings-daily-plan) ページを参照してください。

### 実行計画の削除

この操作は、カレンダーウィジェットの下にあるボタンから、個々の日付および日付範囲に対して実行できます。

- 選択した日の実行計画は、*送信済み*または*完了*オーダーがない場合に削除されます。計画中*のオーダーがある場合は、実行計画と一緒に削除されます。
-[Settings - Daily Plan](/settings-daily-plan) 指定した日の日次計画が削除された場合、日次計画サービスの次の実行では、その日のオーダーが計画されます。
