# レポート - レポートテンプレート

## レポートテンプレート：Top n 失敗ワークフロー

このテンプレートは、失敗したワークフローの実行回数をカウントします。

- 例えば、オーダーがキャンセルされた場合や、[JS7 - Finish Instruction](https://kb.sos-berlin.com/display/JS7/JS7+-+Finish+Instruction) が使用された場合などです。
- ワークフローの実行は、ジョブが失敗したからといって失敗したとはみなされません。例えば、[JS7 - Retry Instruction](https://kb.sos-berlin.com/display/JS7/JS7+-+Retry+Instruction) が使用された場合、そのジョブの再試行は成功します。その代わりに、オーダーの履歴ステータスが考慮されます。

## レポートテンプレート：Top n 失敗ジョブ

このテンプレートは、失敗したジョブの実行回数をカウントします。

- Shell ジョブの実行はジョブの終了コードと、オプションで stderr チャンネルへの出力に基づいて失敗したとみなされます。
- JVM ジョブの実行は、例外を保持できるジョブの結果に基づいて失敗したとみなされます。

## レポートテンプレート：Top n 並列ジョブ実行エージェント

このテンプレートは、エージェント毎のジョブの並列実行をカウントします。以下の場合、Job1 は Job2 と並列であるとみなされます。

- ジョブ2が開始した後、ジョブ2が終了する前にジョブ1が開始した場合。
- ジョブ1がジョブ2の開始後、ジョブ2が終了する前に終了した場合。

## レポートテンプレート：Top n 重要度の高い失敗ジョブ

このテンプレートは、クリティカルなジョブの失敗をカウントします。重要度はジョブの属性です。

カウントは：Top n 失敗ジョブと同様に失敗ジョブの実行回数です。

## レポートテンプレート：Top n オーダーキャンセルされた失敗ワークフロー

このテンプレートは、キャンセルされたオーダーによる失敗ワークフローの回数をカウント します。

*キャンセル* 操作は、ユーザーの操作によってオーダーに適用されます。

## レポートテンプレート：Top n 実行時間の長いワークフロー

このテンプレートでは、成功したワークフローの実行時間を考慮します。失敗したワークフローは考慮されません。

## レポートテンプレート：Top n 実行時間の長いジョブ

このテンプレートは成功したジョブの実行時間を考慮します。失敗したジョブは考慮されません。

## レポートテンプレート：Top n 期間中実行回数の多いジョブ

このテンプレートはレポート期間をステップ毎に分割します。ステップの期間は[JS7 - Reports - Configuration](https://kb.sos-berlin.com/display/JS7/JS7+-+Reports+-+Configuration) の *レポート期間長* 設定によって決定されます。 次のステップの開始はレポート構成の "期間ステップ" 設定によって決定されます。

例 

- ステップの期間：5m
- ステップの重なり: 2m
  - 00:00-00:05
  - 00:02-00:07
  - 00:04-00:09

実行中のジョブ数はステップごとにカウントされます。

## レポートテンプレート：Top n 期間中実行回数の多いワークフロー

このテンプレートでは、*レポート期間*をステップに分割します。ステップの期間は[JS7 - Reports - Configuration](https://kb.sos-berlin.com/display/JS7/JS7+-+Reports+-+Configuration) の *ステップ期間* 設定によって決定されます。 次のステップの開始はレポート構成の *ステップ重複* 設定によって決定されます。

実行中のオーダーの数はステップごとにカウントされます。

## レポートテンプレート：Top n 成功ジョブ

このレポートテンプレートでは、成功ジョブがカウントされます。失敗したジョブは考慮されません。

失敗の原因については、次のテンプレートを参照してください： *Top n 失敗ジョブ* 

## レポートテンプレート：Top n 成功ワークフロー

このテンプレートは、成功したワークフローをカウントします。失敗したワークフローは考慮されません。

ワークフロー失敗の理由については、次のテンプレートをご参照ください：  *Top n 失敗ワークフロー*
