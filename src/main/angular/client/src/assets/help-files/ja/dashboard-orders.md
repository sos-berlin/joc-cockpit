# オーダー

オーダー*パネルには、以下のソースからのオーダーに関する情報が表示されます：

- によって追加されたオーダー。[Daily Plan](/daily-plan)
- [Workflows](/workflows) ビューからユーザーがオンデマンドで追加したオーダー
- [File Order Sources](/configuration-inventory-file-order-sources) から追加されたオーダー。

## オーダー状態

オーダー*パネルは、オーダーの現在の状態に関する情報を提供します。このパネルは、オーダーの状態が変更されると更新されます。

- **保留中] **オーダーは開始時刻を指定せずにワークフローに追加されています。
- **スケジュール中** オーダーはワークフローに追加され、後日実行される予定です。
- **進行中** オーダーはワークフロー指示によって処理されていますが、ジョブは実行されていません。 
- **実行中** オーダーはジョブ実行中です。 
- **中断中** オーダーはユーザーの介入により中断され、再開することができます。
- **完了** オーダーはワークフローを完了しましたが、削除されていません。例えば、ファイルオーダーソースがファイル監視に使用され、ワークフローが受信ファイルを（再）移動しない場合などです。この場合、ファイルが受信ディレクトリに存在する限り、オーダーはそのまま残ります。
- **プロンプト** オーダーは、ワークフローの *プロンプト命令* により保留され、ワークフローの実行を続行するためにユーザーの確認が必要です。
- **待機中** オーダーは、*Resource Lock*、*Notice*、*Retry*、または *Cycle* インターバルなどのリソースを待機します。
- ブロック**された** オーダーは、オーダーが追加されてからエージェントに到達できない場合などに開始できません。
- **失敗** オーダーは、ジョブが失敗したか、*失敗命令*によってオーダーが進行できないことを示します。 

表示された数のオーダーをクリックすると、[Orders Overview](/orders-overview) に移動し、オーダーの詳細が表示されます。

## フィルター

パネルの右上にあるドロップダウンボタンは、日付範囲からオーダーを選択することができます：

- **All** コントローラとエージェントで利用可能なすべてのオーダーを表示します。
- **今日**のオーダーは、ユーザーの[Profile - Preferences](/profile-preferences) のタイムゾーンから計算される現在の日に関連しています。
  - **保留中** オーダーの開始時刻は表示されません、
  - **スケジュール** オーダーは、当日の開始時間を保持しています、
  - **進行中** 過去の日付のオーダー、
  - **実行中** 過去の日付のオーダー、
  - 過去の日付の**中断**オーダー
  - 過去の日付の**完了**オーダー、
  - 過去の日付の**完了**オーダー、
  - **待機中** 過去日付のオーダー
  - 過去の日付の**ブロックされた**オーダー、
  - **失敗** 過去日付のオーダー。
- 次の1時間**には、次の1時間の**スケジュール**されたオーダーが含まれます。
- **次の12時間**には、次の12時間のオーダーが含まれます。
- **次の24時間**には、次の24時間のオーダーが含まれます。
- **翌日**には、翌日末までのオーダーが含まれます。
- **翌7日間**には、翌7日後までのオーダーが含まれます。
