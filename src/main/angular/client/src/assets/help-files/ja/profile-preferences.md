# プロファイル - 設定値

*プロファイル - 設定値* タブでは、ユーザーアカウントの設定を行います。

ユーザーが初めてJOCコックピットに接続すると、*デフォルトユーザーアカウント*の環境設定がユーザーの環境設定にコピーされます。デフォルトユーザーアカウントは[設定 - JOC](/settings-joc) 画面で設定します。

## ロール

ユーザーアカウントに割り当てられたロールが表示されます。パーミッションはロールから割り当てられ、[プロファイル - パーミッション](/profile-permissions) タブから利用可能です。

## 設定値

ユーザーは自由に設定を変更することができます。

### ブラウザ関連の環境設定

このセクションの環境設定は、使用しているブラウザのデフォルト値を使用します：

- **言語** はJOCコックピットのインターフェース言語で、英語、フランス語、ドイツ語、日本語から選択できます。
- **タイムゾーン** は、JOCコックピットに表示される日付が変換されるタイムゾーンを指定します。
- **日付形式** は利用可能な日付フォーマットのリストから選択することができます。

### 関連する環境設定

環境設定は JOCコックピットのリスト表示に適用されます。値を増やす場合は、次のことを考慮する必要があります：

- JOCコックピットからより多くのデータを読み込んでも、GUIの応答性は向上しません。
- リストが長くなると、レンダリングのためにブラウザのメモリとCPUの消費量が増えます。

共通値で管理できる以下の設定は、*グループリミット* リンクから見つけてください：

- **最大履歴表示数** は、[履歴 - オーダー](/history-orders) 画面に適用されます。
- **最大操作ログ表示数** は、[操作ログ](/audit-log) 画面に適用されます。
- **最大通知表示数** は、*モニター - オーダー通知* および *モニター - システム通知* 画面に適用されます。
- **最大オーダー表示数** は、[オーダー履歴](/orders-overview) 画面に適用されます。
- **最大実行計画表示数** は、[実行計画](/daily-plan) 画面に適用されます。
- **最大ワークフローあたりオーダー表示数** は、[ワークフロー](/workflows) 画面で表示可能なオーダーの数を制限します。
- **最大ファイル転送表示数** は、[履歴 - ファイル転送](/history-file-transfers) 画面に適用されます。
- **最大リソースロックあたりオーダー表示数** は、[リソース - ロック](/resources-resource-locks) 画面で表示されるオーダーの数を制限します。
- **最大通知ボードあたりオーダー表示数** は、[リソース - 通知ボード](/resources-notice-boards) 画面で表示されるオーダーの数を制限します。

### ワークフロー画面関連設定

設定は[ワークフロー](/workflows) 画面に適用されます：

- **最大オーダー履歴表示数** は *オーダー履歴* 画面で表示されるオーダー履歴の数を制限します。
- **最大タスク履歴表示数** は、*タスク履歴* 画面で表示されるタスク履歴の数を制限します。
- **対象あたり最大操作ログ行数** は、*操作ログ* 画面で表示される操作ログ数を制限します。

### ジョブ定義 - インベントリー画面関連設定

- **最大お気に入り数** ジョブをエージェントに割り当てる時などに、お気に入りの表示数を制限します。

### ページ行数関連設定

ページ行数の設定はどのページにも適用されます：

- **最大リスト表示数** は、1ページに表示される行数を制限します。
- **ページ行数** は、1ページで表示される行数を制限します。10、25、50、100、1000のうちのデフォルト値を指定します。

### テーマ関連設定

- **テーマ変更** では、テーマを切り替えることができます。いくつかのテーマは、視覚障害のあるユーザーにより適した高コントラストで提供されています。
  - **オーダーステータスの色変更** は、*テーマの変更* の右側にあるアイコンから利用可能で、[オーダーステータス](/order-states) のデフォルト色を変更できます。JS7のドキュメントでは異なる色で表現されている色を変更するのは混乱するかもしれません。しかし、視覚に障害のあるユーザーには、これは便利かもしれません：ユーザーは、オーダーステートに使用される各色のRGB値を指定することができます。
- **ヘッダーカラー** は、*Light* テーマが使用されている場合に利用可能です。メニューバーの背景色を変更することができます。この設定は、例えば、ユーザが開発、テスト、プロジェクトのために別々のJS7環境で作業する場合に適用できます：異なる背景色を使用すると、関連するJS7環境を識別するのに役立ちます。
- **テーマ名**　は、メニューバーのすぐ下に表示されます。*ヘッダーカラー* と同様に、これは関連するJS7環境を識別するために使用することができます。

### エディタ環境関連設定

- **タブサイズ** は、*ジョブスクリプト* を編集するときに[ジョブ定義 - インベントリー - ワークフロー - ジョブプロパティ](/configuration-inventory-workflow-job-properties) タブで使用されます。この設定では、TABキーを押したときのサイズに対応するスペースの数を指定します。

### 表示関連設定

- **ログ表示** は、[オーダーログ](/order-log) および[タスクログ](/task-log) の表示方法を設定します。どちらのログ表示画面でも、ログの表示とダウンロードが可能です。
- **ドキュメント表示** は、ワークフローとジョブのユーザードキュメントの表示方法を設定します。

### ジョブ定義画面関連の環境設定

- **サブフォルダー/フォルダーコンテンツ表示** は、*ジョブ定義-&gt;インベントリー*　表示 *ナビゲーションパネル*　でフォルダーをクリックしたときの動作を、利用可能なオブジェクトのみを表示するか、利用可能なオブジェクトとサブフォルダーを表示するかを指定します。

### その他環境設定

- **ファイル転送表示** 各ファイル転送が予測可能な数のファイルを含む場合など、1回の転送に数千のファイルが含まれる場合は、この設定を無効にした方がよいでしょう。
- **操作ログコメント表示** は、オーダーの追加やキャンセル、ワークフローの一時停止など、オブジェクトを変更する際にユーザーに理由を指定させます。このユーザー設定は、関連する[設定 - JOC](/settings-joc) から上書きできます。
- **ログタイムスタンプにタイムゾーンを使用** は、エージェントが異なるタイムゾーンまたはコントローラーサーバーのタイムゾーンと異なるサーバーで実行されている場合に適用されます。このような状況では、おそらく異なるエージェントで実行された多数のジョブのログ出力を保持するオーダーログが混乱する可能性があります。この設定は、ログのタイムスタンプをユーザーのプロファイルで指定された *タイムゾーン* に変換します。
- **現在コントローラー** JOCコックピットに複数のコントローラーが接続されている場合適用されます。[履歴 - オーダー](/history-orders) ]画面など、多くの表示で提供されているオプションです。チェックを入れると、現在選択されているコントローラーに送信されたオーダーに限定し、それ以外の場合は接続されているすべてのコントローラのオーダーを表示します。この設定は、JOCコックピットビューの関連する *現在コントローラー* オプションのデフォルト値を決定します。
- **インベントリーオブジェクトのツールチップ抑止** は、[ジョブ定義 - インベントリー - ワークフロー - ジョブプロパティ](/configuration-inventory-workflow-job-properties)画面のツールチップ表示に関連します。ツールチップは、入力フィールドのラベルにマウスを移動するとポップアップ表示され、可能な入力を説明することでユーザーを支援します。これはJS7にあまり慣れていないユーザーには便利ですが、経験豊富なユーザーにはツールチップは必要ないかもしれません。
- **ライセンス警告を確認** は、通常1年間に制限されているサブスクリプション・ライセンスの使用について言及しています。ライセンスの有効期限が切れる前に、JOCコックピットは関連する警告を表示します。ユーザーは関連するライセンス期限切れの警告を抑制することができます。詳細は[JS7 - How to manage License Expiration Warnings](https://kb.sos-berlin.com/display/JS7/JS7+-+How+to+manage+License+Expiration+Warnings) を参照してください。
- **オプション表示** は、*ジョボ定義-&gt;インベントリー* 表示の関連スライダーボタンを有効にします。例えば、[ジョブ定義 - インベトリー - ワークフロー - ジョブオプション](/configuration-inventory-workflow-job-options) タブで、ジョブ定義のより詳細なオプションを提供します。
- **変数リストをたたむ** は、[ジョブ定義 - インベントリー - ワークフロー](/configuration-inventory-workflows) の表示に適用され、多数のデータ型からワークフロー変数を指定できます。*データ型（配列）* が使用される場合、より多くのエントリを保持することができます。ワークフローを編集する際、リスト変数を即座に展開したくない場合有効にします。

### 表示形式の設定

- **表示形式** は、ウィンドウの右上隅に関連インジケータを表示する多くのビューに適用されます。この設定では、デフォルトで使用される表示形式を指定します。ユーザーは、どの画面でも必要に応じて表示形式を変更できます。*カード表示* は、*リスト表示* 形式よりも画面上に多くのスペースを必要とします。しかし、カードからの可視性を好むユーザーもいるでしょう。
- **オーダー表示** は、*表示形式*設定と似ていますが、[オーダー](/orders-overview) 表示に適用されます。さらに、この設定には *一括* 形式があり、一括操作からオーダーを移行することができます。

### ワークフローレイアウトカスタマイズ

[ジョブ定義 - インベトリー - ワークフロー](/configuration-inventory-workflows) 表示でのワークフローの表示に適用されます：

- **向き** ワークフローの表示を縦表示と横表示に切り替えることができます。ワークフローを縦長・横長に表示させることができます。
- **隣接するレイヤーの命令間の間隔**は、垂直方向のワークフロー指示間の間隔を変更することができます。
- **同じレイヤーの命令間の間隔**は、水平方向のワークフロー命令間の間隔を変更することができます。
- **ラウンドエッジ接続** ジョブなどのワークフロー命令の表示を曲線にします。
