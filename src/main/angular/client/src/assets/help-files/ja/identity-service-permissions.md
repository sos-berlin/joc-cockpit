# アイデンティティサービス許可

アイデンティティサービスは、認証と承認によってJOCコックピットへのアクセスを制御します。[Identity Services](/identity-services) を参照してください。

認証のために、JS7 は役割ベースのアクセスモデル（RBAC）を提供します。

- ロールは利用可能な権限から自由に設定できます、
- ユーザは1つまたは複数のロールを割り当てられ、それらのロールは結果として得られる権限にマージされます。

パーミッションは以下の状態のいずれかを指定します：

- パーミッションが割り当てられていない状態 (背景色が白)、
- パーミッションは許可されています (背景色は青)、
- 許可されていない (背景色が灰色)。

パーミッションは、ユーザアカウントの結果的なパーミッションに対して、すべてのロールからマージされます。

## フォルダのスコープ

ロールのパーミッションの範囲は、1つ以上のインベントリフォルダに制限することができます。

- ユーザーはインベントリフォルダを選択し、再帰的な使用を指定するために、ビューの右上にある*フォルダを追加*ボタンを適用できます。
- ユーザはロールにインベントリ・フォルダをいくつでも追加できます。

## 権限ツリー

パーミッションは枝分かれした階層を提供するツリーと考えることができます。より高いレベルでのパーミッションの付与または拒否は、ツリーのより深いレベルに再帰的にパーミッションの割り当てを継承します。

### パーミッションの付与と拒否

パーミッションはバッテリーに似た長方形で可視化されます：

- バッテリーの右側にあるポールをクリックすると、下位のパーミッションが展開/折りたたみされます。
- バッテリーの背景をクリックすると、パーミッションが未割り当てのステータスと付与されたステータスに切り替わります：
  - 背景色が白の長方形は未割り当ての許可を示します。
  - 青い背景色の長方形は、許可されたパーミッションであり、子孫のパーミッションに渡されます。<br/><img src="identity-service-permissions-granted.png" alt="Granted Permissions" width="600" height="100" />
  - 背景色が水色の長方形は、継承されたパーミッションであることを示します。パーミッションの変更には、親パーミッションを付与するのではなく、子パーミッションを個別に付与する必要があります。 <br/><img src="identity-service-permissions-inherited.png" alt="Inherited Permissions" width="600" height="100" />
- パーミッションの矩形内の+アイコンをクリックすると、パーミッションは背景色が灰色で示される拒否ステータスに切り替わります。拒否されたパーミッション内の-アイコンをクリックすると、背景色が白の未割り当てパーミッションになります。<br/><img src="identity-service-permissions-denied.png" alt="Denied Permissions" width="600" height="100" />

### パーミッションの折りたたみと展開

以下のボタンはパーミッションを展開/折りたたむために提供されます：

- **Expand All**, **Collapse All** はすべてのパーミッションを展開または折りたたみます。
- **Expand Active** は許可/拒否されたパーミッションを展開し、継承されたパーミッションは折りたたまれたままにします。
- **未割り当てのパーミッションを折りたたみます。

## グラフィカル表示と表形式表示

右上隅にはパーミッションを表示するための以下のボタンがあります：

- **グラフィカルビュー** は、バッテリーの形状を使用したツリーからパーミッションを表示します。
- **Tabular View** 許可をコロンで区切られたテキストで表示します。
