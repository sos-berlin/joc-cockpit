# ジョブ定義 - インベントリー - 操作 - オブジェクトの配置

オブジェクトの配置には、オブジェクトを1つまたは複数のコントローラーに転送することが含まれます。これは *コントローラー* システムフォルダーから利用可能なワークフローやジョブリソースなどのオブジェクトに適用されます。

*ジョブ定義 -&gt; インベントリー* 画面は、単一のオブジェクトの配置とフォルダーからのオブジェクトの配置を提供します。[フォルダーの配置](/configuration-inventory-operations-deploy-folder) を参照してください。

*配置* ボタンから単一のオブジェクトを配置する場合、以下のようなポップアップウィンドウが表示されます：

<img src="deploy-workflow.png" alt="Deploy Workflow" width="600" height="460" />

## コントローラーへの配置

入力フィールド **コントローラー** には、オブジェクトを配置するコントローラーを指定します。

デフォルトでは、現在選択されているコントローラーが使用されます。

## 実行計画更新

ワークフロー、スケジュール、カレンダーなどのオブジェクトの展開は、[実行計画](/daily-plan) に影響を与えます。通常オブジェクトの最新バージョンが実行計画のオーダーに使用されます。ユーザーは以下のオプションから動作を制御します：

- **実行計画更新**
  - **Now** は、開始する時点でスケジュールされているオーダーの実行計画を更新します。
  - **開始位置** を選択すると、実行計画を更新する開始日の入力フィールドが追加されます。
  - **No** を選択すると、実行計画は更新されません。既存のオーダーは、以前に配置されているバージョンのオブジェクトを使用します。
- **本日の最新オーダーを含む** にチェックを入れると、現在の日付で過去の時間にスケジュールされ多物の、遅延のため開始されていないオーダーが含まれます。

## 依存関係を含める
インベントリーオブジェクトは依存関係によって関連付けられます。[依存関係](/dependencies-matrix) を参照してください。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトを配置する際には、整合性が考慮されます：

- 例えば、ジョブリソースが作成され、新しく作成されたワークフローを参照している場合、ワークフローの配置はジョブリソースの配置も含みます。
- 配置済のワークフローによって参照されるジョブリソースを取消または削除する場合、対応するワークフローも取消または削除されなければなりません。

ユーザーは以下のオプションから一貫性のある配置を管理します：

- **依存関係を含む** をチェックすると、参照元オブジェクトと参照先オブジェクトの両方が含まれます。
    - 関連するオブジェクトが下書きステータスの場合、共通の配置が提供されます。このオプションは、オブジェクトの関係の変更によって必要になった場合に強制されます。
    - 関連オブジェクトが配置済み/リリース ステータスの場合、共通配置はオプションです。ユーザーは、関連オブジェクトを選択して共通ディプロイメントを行うことができます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、配置/リリースされているかどうかを確認する必要があります。一貫性のない配置によってオブジェクトが見つからない場合、コントローラはエラーメッセージを表示します。
