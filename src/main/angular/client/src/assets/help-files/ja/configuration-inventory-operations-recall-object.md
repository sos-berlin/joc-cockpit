# ジョブ定義 - インベントリー - 操作 - オブジェクトを下書きに戻す

オブジェクトを下書きに戻すは、例えば[実行計画](/daily-plan) で使用するために、オブジェクトを下書きに戻します。 これは、*実行計画* システムフォルダーから利用可能なスケジュールやカレンダーなどのオブジェクトに適用されます。

*ジョブ定義-&gt;インベントリー* 画面では、単一のオブジェクトとフォルダーからのオブジェクトの下書きに戻すことが可能です（[ジョブ定義 - インベントリー - 操作 - フォルダーを下書きに戻す](/configuration-inventory-operations-recall-folder) を参照してください。

ナビゲーションパネルのオブジェクトのアクションメニューから利用可能な *下書きに戻す* 操作から単一のオブジェクトを下書きに戻す場合、以下のようなポップアップウィンドウが表示されます：

<img src="recall-schedule.png" alt="Recall Schedule" width="600" height="300" />

## 実行計画更新

スケジュールやカレンダーなどのオブジェクトの下書きに戻すは[実行計画](/daily-plan) に影響します。 
関連するスケジュールが参照するワークフローの既存のオーダーは、コントローラーから取消され、実行計画から削除されます。

## 依存関係を含む

インベントリーオブジェクトは依存関係によって関連付けられます（[依存関係](/dependencies-matrix) を参照）。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトを下書きに戻す際には、一貫性が考慮されます：

- 例えば、スケジュールが作成され、新しく作成されたカレンダーを参照している場合、スケジュールのリリースはカレンダーのリリースも含みます。スケジュールが作成され、新しく作成されたカレンダーを参照する場合、スケジュールのリリースはカレンダーのリリースも含みます。
- リリース済みのスケジュールが参照しているカレンダーを取消または削除する場合、スケジュール自体も取消または削除する必要があります。これにはスケジュールが参照しているワークフローの取消または削除も含まれます。

ユーザーは以下のオプションから一貫性のある展開をコントロールします：

- **依存関係を含む**  をチェックすると、参照元オブジェクトと参照先オブジェクトの両方が含まれます。
    - 関連するオブジェクトが下書きステータスの場合、共通で下書きに戻されます。このオプションは、オブジェクトの関係の変更によって必要な場合、強制されます。
    - 関連オブジェクトが配置済み/リリース ステータスの場合、共通下書きに戻すはオプションです。ユーザーは、関連オブジェクトを選択して共通しg￥た書に戻すを行うことができます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、配置/リリースされているかどうかを確認する必要があります。一貫性のない配置によってオブジェクトが見つからない場合、コントローラーはエラーメッセージを表示します。
