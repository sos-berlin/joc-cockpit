# 承認者プロファイル

[JS7 - Approval Process](https://kb.sos-berlin.com/display/JS7/JS7+-+Approval+Process) は、オーダーの追加やキャンセルなど、セカンドユーザの承認が必要な操作のために提供されます。これには、スケジュールオブジェクトを変更する操作も含まれます。

承認プロセスには、以下の役割が含まれます：

- 要求者*は、承認が必要な操作の実行を要求します。
- 承認者*は、承認要求を確認または拒否します。

承認プロセスの基本機能は次のとおりです：

- 4-eyesの原則を実装するため： *Approver*は、介入を実行する前に、*Requestor*のアカウント、ロール、および権限の範囲で介入を確認する必要があります。 
- 保留中の承認リクエストを追跡します。
- 複数の *Approvers* にフォールバックを提供するため。

## 承認者プロファイルの一覧

承認者プロファイルを追加するには、ビューの右上にある関連ボタンをクリックします。

承認者プロファイルの一覧には、次のプロパティが表示されます：

- **承認者プロファイルの一覧には、次のプロパティが表示されます。
- **承認者の姓**、名**を指定します。
- **E-Mail**は*Approver*のE-mailアドレスを指定します。指定した場合、*Approver* は[Approval Requests](/approval-requests) の受信通知をメールで受け取ります。

## 承認者プロファイルの操作

承認者プロファイルの3点アクションメニューで、次の操作を行います：

- **承認プロファイルのプロパティを更新します。
- **承認プロファイルのプロパティを更新します。
