# ワークフローへのオーダー追加

オーダーはオンデマンドで追加でき、実行計画とは別に実行されます。

デフォルト値で満足し、即座に実行するオーダーを送信したいユーザーは、これ以上入力を追加する必要はありません。 

### オーダー属性

- **オーダー名**：オーダー名**: 多くのビューでオーダーをフィルターするために使用できるオプションの名前です。
- **タグ名**：オーダーに追加されるタグをいくつでも指定できます。タグは[Settings - JOC Cockpit](/settings-joc) ページから指定した場合、複数のビューに表示されます。
- **Ignore Job Admission Times**：ジョブは特定の曜日やタイムスロットに限定することができます。タイムスロット外に到着したオーダーは次のタイムスロットを待たなければなりません。このオプションにより、ジョブはそのような制限から独立して開始されます。

### 開始時間

- **今すぐ**：オーダーはすぐに開始されます。
- **特定の日付**：指定日時**：オーダーは指定日時に開始されます。
- **現在時刻からの相対時間**：時、分、秒のオフセットでオーダーを開始します。例
  - **30s**:30秒後
  - **15m**:15分後
  - **1h**:1時間後
  - **1時間15分30秒**または**01時15分30秒**：1時間15分30秒後
- **開始時間なし**：開始時間なし**：オーダーは開始されませんが、*保留*状態から利用可能になり、後で開始時間を割り当てることができます。

### オーダーの依存関係

- **通知スペースキー**：ワークフローが通知に基づく依存関係を保持している場合、依存関係を解決する実行 計画日を指定することができます。デフォルトでは現在日が使用されます。
  - プランがオープンされた過去の日付が使用できます。
  - 未来の日付が使用されます。

### オーダーポジション

オーダーをワークフローの最初のノードから開始しない場合は、位置を指定できます。

- **ブロック位置**：Try/Catch、ResourceLock、Fork/Join などのブロック命令を保持するワークフローでは、 関連する命令を選択できます。
- **開始位置**：開始位置が指定されていない場合、オーダーは最初のノードから開始されます。
  - ブロック位置が指定されていない場合、ワークフロー内のトップレベルの命令を選択することができ、そこからオーダーを開始します。
  - ブロック位置が指定されている場合、開始位置はブロック内の同じレベルのノードになります。
- **終了位置**：
  - ブロックポジションが指定されていない場合、ワークフロー内のトップレベルの命令が選択可能です。
  - ブロックポジションが指定された場合、オーダーが終了する前に、ブロック内の任意の同レベルノードを指定することができます。
  - 終了位置は複数指定可能。
- **優先度**：
  - 優先順位(**Priority**): オーダーがワークフロー内で並列性を制限するResource Lock命令を満たす場合、その *優先順位(*Priority*)は、待機中の*オーダーのキュー内での位置を決定します。
  - *優先度*は負、0、正の整数、またはショートカットから指定します。優先度*は高い方が優先されます。ショートカットには以下の値があります：
    - **Low**： -20000
    - **通常**以下： -10000
    - **通常**：0
    - **標準以上10000
    - **高い**：20000

### オーダーのパラメータ化

- **スケジュールからパラメータを割り当てる**：Assign Parameterization from Schedule**： ワークフローにスケジュールが割り当てられている場合、変数やタグな どのパラメータ設定を現在のオーダーにコピーします。
- **変数の変更**： 
  - ワークフローがデフォルト値のない変数を指定している場合、現在のオー ダーは関連する値を指定する必要があります。
  - ワークフローがデフォルト値を持つ変数を指定している場合、このリンクから新しい値を指定する変数を選択できます。
- **変数の変更**：変数の変更**：*変数の変更*と同様に動作しますが、利用可能なすべての変数を表示します。

### 追加オーダー

- **オーダーの追加**：複数のオーダーをワークフローに追加する場合、このリンクは追加オーダーのパラ メータ設定を追加します。
- **スケジュールからオーダーを追加**：ワークフローに1つ以上のスケジュールが割り当てられている場合、各スケジュールにスケジュールからパラメータ化されたオーダーが追加されます。
