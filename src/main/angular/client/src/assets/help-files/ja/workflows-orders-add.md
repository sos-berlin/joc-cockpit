# ワークフローへのオーダー追加

オーダーをオンデマンドで追加することによって、実行計画とは別にワークフローを実行します。

デフォルト値で即座に実行するオーダーを追加したいユーザーは、これ以上入力する必要はありません。 

### オーダー属性

- **オーダー名**: 多くのビューでオーダーを検索するために使用できるオプションの名前です。
- **タグ名**：オーダーに追加されるタグをいくつでも指定できます。タグ数は[設定 - JOC](/settings-joc) ページから指定した数だけ表示されます。
- **許可時間を無視**：ジョブ実行時間は特定の曜日や時間帯に限定することができます。許可時間外のオーダーは次の許可時間まで待たなければなりません。このオプションにより、ジョブはそのような制限から関係なく開始できます。

### 開始時刻

- **現在**：オーダーはすぐに開始されます。
- **特定日付**：オーダーは指定日時に開始されます。
- **相対時刻**：時、分、秒でオーダーを開始します。例
  - **30s**:30秒後
  - **15m**:15分後
  - **1h**:1時間後
  - **1h15m30s**または**01:15:30**：1時間15分30秒後
- **未定**：オーダーは開始されませんが、*保留*状態から利用可能になり、後で開始時間を割り当てることができます。

### オーダーの依存関係

- **通知予定日**：ワークフローに通知ボードに基づく依存関係がある場合、依存関係を解決する実行計画日を指定することができます。デフォルトでは現在日が使用されます。
  - 実行計画が登録された過去の日付が使用できます。
  - 未来の日付も使用できます。

### オーダー位置

オーダーをワークフローの最初のノードから開始しない場合は、位置を指定できます。

- **ブロック位置**：Try/Catch、ResourceLock、Fork/Join などのブロック命令を含むワークフローでは、 関連する命令を選択できます。
- **開始ノード**：開始位置が指定されていない場合、オーダーは最初のノードから開始されます。
  - ブロック位置が指定されていない場合、ワークフロー内のトップレベルの命令を選択することができ、そこからオーダーを開始します。
  - ブロック位置が指定されている場合、開始位置はブロック内の同じレベルのノードになります。
- **終了ノード**：
  - ブロックポジションが指定されていない場合、ワークフロー内のトップレベルの命令が選択可能です。
  - ブロックポジションが指定された場合、オーダーが終了する前に、ブロック内の任意の同レベルノードを指定することができます。
  - 終了位置は複数指定可能です。
- **優先度**: オーダーがワークフロー内で並列性を制限するResource Lock命令を含む場合、*優先度*は、待機中のオーダーのキュー内での位置を決定します。
  - *優先度*は負、0、正の整数、またはショートカットから指定します。*優先度*は高い方が優先されます。ショートカットには以下の値があります：
    - **低**： -20000
    - **中低**： -10000
    - **中**：0
    - **高中**：10000
    - **高**：20000

### オーダーの変数設定

- **スケジュールから変数設定**： ワークフローにスケジュールが割り当てられている場合、変数やタグなどの変数設定を現在のオーダーに適用します。
- **変数の変更**： 
  - ワークフローがデフォルト値のない変数を指定している場合、現在のオーダーは関連する値を指定する必要があります。
  - ワークフローがデフォルト値を持つ変数を指定している場合、このリンクから新しい値を指定する変数を選択できます。
- **変数の変更**：*変数の変更*と同様に動作しますが、利用可能なすべての変数を表示します。

### オーダー追加

- **オーダー追加**：複数のオーダーをワークフローに追加する場合、このリンクは追加オーダーの変数設定を追加します。
- **全スケジュールからオーダー追加**：ワークフローに1つ以上のスケジュールが割り当てられている場合、各スケジュールにスケジュールからパラメータ化されたオーダーが追加されます。
