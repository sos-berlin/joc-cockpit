# ジョブ定義 - インベントリー - 操作 - フォルダーのリリース

オブジェクトのリリースは、例えば[実行計画](/daily-plan) で使用するために、オブジェクトを有効にします。 これは、*実行計画* システムフォルダーから利用可能なスケジュールやカレンダーなどのオブジェクトに適用されます。

*ジョブ定義-&gt;インベントリー* 画面では、単一のオブジェクトをリリースできます。[ジョブ定義 - インベントリー - 操作 - オブジェクトのリリース](/configuration-inventory-operations-release-object)を参照してください。

フォルダーのアクションメニューから *リリース* ボタンを使用してフォルダーからオブジェクトをリリースする場合、次のようなポップアップウィンドウが表示されます：

<img src="release-folder.png" alt="Release Folder" width="592" height="560" />

## 実行計画更新

スケジュールやカレンダーなどのオブジェクトのリリースは、[実行計画](/daily-plan) に影響を与えます。 通常、実行計画のオーダーには最新バージョンのオブジェクトを使用する必要があります。ユーザーは以下のオプションから動作を制御できます：

- **実行計画更新**
  - **Now** は、開始する時点でスケジュールされているオーダーの実行計画を更新します。
  - **開始位置** を選択すると、実行計画を更新する開始日の入力フィールドが追加されます。
  - **No** を選択すると、実行計画は更新されません。既存のオーダーは、以前に配置されているバージョンのオブジェクトを使用します。
- **本日の最新オーダーを含む** にチェックを入れると、現在の日付で過去の時間にスケジュールされ多物の、遅延のため開始されていないオーダーが含まれます。
 
## フォルダーないにも適用

**フォルダー内にも適用** オプションを使用すると、サブフォルダーのオブジェクトを再帰的にリリースすることができます。

## 依存関係を含む

インベントリーオブジェクトは依存関係によって関連付けられます。[依存関係](/dependencies-matrix) を参照してください。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトをリリースする際には、整合性が考慮されます：

- 例えば、スケジュールが作成され、新しく作成されたカレンダーを参照している場合、スケジュールのリリースはカレンダーのリリースも含みます。スケジュールが作成され、新しく作成されたカレンダーを参照する場合、スケジュールのリリースはカレンダーのリリースも含みます。この処理には、スケジュールで参照されている下書きのワークフローの配置も含まれます。
- リリース済みのスケジュールで参照されているカレンダーを取り消しまたは削除する場合、対応するスケジュールも同様に取り消しまたは削除する必要があります。これにはスケジュールが参照しているワークフローの取り消しまたは削除も含まれます。

ユーザーは以下のオプションから一貫性のある展開をコントロールします：

- **依存関係を含む**  をチェックすると、参照するオブジェクトと参照されるオブジェクトの両方が含まれます。
    - 関連するオブジェクトが下書きステータスの場合、共通で配置されます。このオプションは、オブジェクトの関係の変更によって必要な場合、強制されます。
    - 関連オブジェクトが配置済み/リリース ステータスの場合、共通配置はオプションです。ユーザーは、関連オブジェクトを選択して共通ディプロイメントを行うことができます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、配置/リリースされているかどうかを確認する必要があります。一貫性のない配置によってオブジェクトが見つからない場合、コントローラーはエラーメッセージを表示します。
