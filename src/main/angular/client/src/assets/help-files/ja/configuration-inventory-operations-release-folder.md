# 設定 - インベントリ - オペレーション - リリースフォルダ

オブジェクトのリリースは、例えば[Daily Plan](/daily-plan) で使用するために、オブジェクトをアクティブにします。 これは、*Automation* システムフォルダから利用可能なスケジュールやカレンダーなどのオブジェクトに適用されます。

Configuration-&gt;Inventory*]ビューでは、単一のオブジェクトをリリースできます。[Configuration - Inventory - Operations - Release Object](/configuration-inventory-operations-release-object) 、フォルダからオブジェクトをリリースするを参照してください。

フォルダの 3 つの点のアクションメニューから関連する *Release* 操作を使用してフォルダからオブジェクトを解放する場合、次のようなポップアップウィンドウが表示されます：

## 実行計画の更新

スケジュールやカレンダーなどのオブジェクトのリリースは、[Daily Plan](/daily-plan) に影響を与えます。 頻繁に、オブジェクトの更新されたバージョンが実行計画のオーダーに使用されます。ユーザーは以下のオプションから動作を制御します：

- **実行計画を更新します。
  - **Now**は、現在から開始する時点でスケジュールされているオーダーの実行計画を更新します。
  - **Start Date** を選択すると、実行計画を更新する開始日の入力フィールドが追加されます。
  - **いいえ] を選択すると、実行計画は更新されません。既存のオーダーは、以前にデプロイされたバージョンのオブジェクトを使用します。
- **今日の遅れたオーダーを含める]**にチェックを入れると、現在日中の過去の時間にスケジュールされ、遅れて開始されなかったオーダーが含まれます。

## サブフォルダーを含む

Handle recursively** オプションを使用すると、サブフォルダのオブジェクトを再帰的に解放することができます。

## 依存関係を含む

インベントリオブジェクトは依存関係によって関連付けられます。[Dependency Matrix](/dependencies-matrix) を参照してください。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトをリリースする際には、例えば、整合性が考慮されます：

- 例えば、スケジュールが作成され、新しく作成されたカレンダーを参照している場合、スケジュールのリリースはカレンダーのリリースも含みます。スケジュールが作成され、新しく作成されたカレンダーを参照する場合、スケジュールのリリースはカレンダーのリリースも含みます。
- スケジュールが作成され、新しく作成されたカレンダーを参照する場合、スケジュールをリリースすることは、スケジュールによって参照されるワークフローのドラフトをデプロイすることを含みます。これには、スケジュールによって参照されるワークフローを取り消したり、削除したりすることも含まれます。

ユーザーは以下のオプションから一貫性のある展開をコントロールします：

- **依存関係**を含める
  - をチェックすると、参照するオブジェクトと参照されるオブジェクトの両方が含まれます。
    - 関連するオブジェクトがドラフト ステータスの場合、共通のデプロイメントが提供されます。このオプションは、オブジェクトの関係の変更によって必要になった場合に強制されます。
    - 関連オブジェクトが配置済み/リリース ステータスの場合、共通配置はオプションです。ユーザーは、関連オブジェクトを選択して共通ディプロイメントを行うことができます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、デプロイ/リリースされているかどうかを確認する必要があります。一貫性のないデプロイによってオブジェクトが見つからない場合、コントローラはエラーメッセージを表示します。
