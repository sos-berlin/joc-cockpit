# ジョブ定義 - インベントリー - 操作 - フォルダー取り消し

オブジェクトの取り消しは、コントローラーからオブジェクトを削除することと、インベントリーでオブジェクトを下書きステータスに保つことが含まれます。これは *コントローラー* システムフォルダーから利用可能なワークフローやジョブリソースなどのオブジェクトに適用されます。

*ジョブ定義-&gt;インベントリー* 画面では、1 つのオブジェクトの取り消し（[ジョブ定義 - インベントリー - 操作 - オブジェクトの取り消し](/configuration-inventory-operations-revoke-object) を参照）と、フォルダーからのオブジェクトの取り消しができます。

ナビゲーションパネルのフォルダーのアクションメニューから利用可能な *取り消し* 操作を使用してフォルダーからオブジェクトを取り消すと、次のようなポップアップウィンドウが表示されます：

<img src="revoke-folder.png" alt="Revoke Folder" width="600" height="580" />

## コントローラー

この入力フィールドには、オブジェクトを取り消すコントローラーを1つ以上指定します。
デフォルトでは、現在選択されているコントローラーが表示されます。

## 実行計画の更新

ワークフローやジョブリソースなどのオブジェクトの取り消しは[実行計画](/daily-plan) に影響します。 
関連するワークフローの既存のオーダーはコントローラーから呼び出され、実行計画から削除されます。

## サブフォルダーを含む

**フォルダー内にも適用** をチェックすると、サブフォルダーのオブジェクトを再帰的に取り消すことができます。

## 依存関係を含む

インベントリーオブジェクトは依存関係によって関連付けられます。[依存関係](/dependencies-matrix) を参照してください。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトを取り消しする場合、整合性が考慮されます：

- ジョブリソースがワークフローから参照されている場合、ジョブリソースの取り消しはワークフローの取り消しも含みます。
- ワークフローが取り消された場合、ワークフローを参照しているスケジュールは下書きの戻され、関連するオーダーは実行計画から削除されます。

ユーザーは、以下のオプションからオブジェクトの一貫した失効を制御します：

- **依存関係を含む** をチェックすると、参照するオブジェクトと参照されるオブジェクトの両方が含まれます。
    - 関連するオブジェクトが以前に配置またはリリースされている場合、共通でより消されます。オブジェクトの関係で必要であれば、取り消しが強制されます。
    - これは、以前に配置またはリリースされたドラフト状態のオブジェクトにも同様に適用されます。
  - チェックを外すと、依存関係は考慮されません。関連するオブジェクトが有効で、配置/リリースされているかどうかは、 ユーザが確認する必要があります。コントローラーは、オブジェクトが見つからない場合にエラーメッセージを表示します。
