# 承認依頼

[JS7 - Approval Process](https://kb.sos-berlin.com/display/JS7/JS7+-+Approval+Process) は、オーダーの追加やキャンセルなど、他のユーザーの承認が必要な操作のために提供されます。これには、スケジュールオブジェクトを変更する操作も含まれます。

承認管理には、以下が含まれます：

- *承認依頼者* は、承認が必要な操作の承認を依頼します。
- *承認者* は、承認依頼を承認、または拒否します。

承認管理の基本機能は次のとおりです：

- 4-eyesの原則（２名認証）のために、*承認者* は、*承認依頼者* が操作を実行する前に、*承認依頼者* のアカウント、ロール、および権限の範囲を確認する必要があります。 
- 保留中の承認依頼を追跡します。
- 複数の *承認者* の代替候補を複数提供します。

## 承認依頼のリスト

承認依頼は、承認を要求するユーザーによって追加されます。[承認依頼](/approval-request) を参照してください。

承認リクエストのリストは以下のプロパティで提供されます：

- **依頼日時** は、[承認依頼](/approval-request) が追加された時点です。
- **タイトル** は、承認依頼を追加するときに *依頼者*が指定します。
- **依頼元** は、承認依頼を発行したユーザーアカウントを示します。
- **依頼ステータス** は *依頼済*、*承認済*、*取り下げ済*、*実行済* のいずれかです。
- **承認者** は、優先する *承認者* の *姓名* です。
- **承認ステータス** は *保留*、*承認*、*却下* のいずれかです。
- **承認日時** は、*承認者* が承認または却下など、承認リクエストに対応した最新の時点です。
- **依頼URL** は、*依頼者* が使用したい[REST Web Service API](/rest-api) エンドポイントです。
- **種別** は、依頼の範囲を示します。たとえば、コントローラーや実行計画などを対象としています。
- **理由** は、承認リクエストの目的について *依頼者* の説明を示します。
