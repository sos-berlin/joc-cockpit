# ジョブ定義 - インベントリー - 操作 - フォルダー削除

オブジェクトの削除には、コントローラーとインベントリからの削除が含まれます。これは、*コントローラー* および *実行計画* システムフォルダーから利用可能なワークフローやスケジュールなどのオブジェクトに適用されます。

フォルダーの削除には、サブフォルダーを再帰的に削除することも含まれます。削除されたオブジェクトは、インベントリのゴミ箱で引き続き使用できます。

*ジョブ定義-&gt;インベントリー* 画面では、単一のオブジェクトの削除（[ジョブ定義 - インベントリー - 操作 - オブジェクト削除](/configuration-inventory-operations-remove-object) を参照）、およびフォルダーからのオブジェクトの削除が可能です。

ナビゲーションパネルのフォルダーののアクション・メニューから利用できる *削除* 操作を使用してフォルダーを削除する場合、次のようなポップアップウィンドウが表示されます：

## コントローラーからのオブジェクトの削除

オブジェクトを削除すると、そのオブジェクトが配置されているすべてのコントローラーから削除されます。

## 実行計画の更新

ワークフローやスケジュールなどのオブジェクトの削除は、[実行計画](/daily-plan) に影響します。 

関連するワークフローの既存のオーダーはコントローラーからキャンセルされ、実行計画から削除されます。

## 依存関係を含む

インベントリーオブジェクトは依存関係によって関連付けられます（[依存関係](/dependencies-matrix) を参照）。例えば、ジョブリソースとリソースロックを参照するワークフロー、カレンダーと 1 つ以上のワークフローを参照するスケジュールなどです。

オブジェクトを削除する場合、整合性が考慮されます：

- ジョブリソースがワークフローから参照されている場合、ジョブリソースの削除はワークフローの削除を含みます。
- ワークフローが削除された場合、ワークフローを参照するスケジュールは下書きに戻され、関連するオーダーはキャンセルされ、実行計画から削除されます。

ユーザーは以下のオプションからオブジェクトの一貫した削除を制御します：

- **依存関係を含む** をチェックすると、参照するオブジェクトと参照されるオブジェクトの両方が含まれます。
    - 関連オブジェクトが以前に配置またはリリースされている場合、共通で削除/取消しされます： *削除* 操作が実行されたオブジェクトは削除され、関連オブジェクトは取消し/下書きに戻されます。オブジェクトの関係によって必要であれば、取り消しは強制されます。
    - これは、以前に配置またはリリースされた下書きステータスのオブジェクトにも同様に適用されます。
  - チェックを外すと、依存関係は考慮されません。関連するオブジェクトが有効で、配置/リリースされているかどうかは、 ユーザが確認する必要があります。コントローラーは、オブジェクトが見つからない場合にエラーメッセージを表示します。
