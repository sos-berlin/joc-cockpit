# オーダーログビュー

オーダーログビュー*では、2-3秒ごとに更新される実行中のログを見ることができます。これにより、オーダーによって実行されたジョブやワークフロー指示の出力をほぼリアルタイムで追うことができます。

## 出力フィルタリング

オーダーログビューでは、ウィンドウ上部から様々な条件によるフィルタリングが可能です：

- **Main**は、オーダー開始、ジョブ開始、ジョブ開始のパラメーターに関する詳細が表示されることを指定します。このような出力は[MAIN]修飾子で示されます。
- **Success**イベントは[SUCCESS]修飾子で表示され、ジョブが完了したときの結果のパラメータ化などの詳細を表示します。
- **stdout**は、ジョブが標準出力チャネルに書き込んだ出力を表示するかどうかを[STDOUT]修飾子で指定します。
-[JS7 - Job API](https://kb.sos-berlin.com/display/JS7/JS7+-+Job+API)**このようなジョブには、*debug*または*trace*値を保持する*log_level*引数を追加することができます。デバッグ出力がジョブから利用できる場合、[DEBUG]修飾子を使用して表示されます。

ジョブが標準エラーチャンネルに書き込んだログ出力はフィルタリングの対象にはならず、[STDERR] 修飾子を使って表示されます。

## ログ出力の表示

ログ出力は到着順に表示されます。ワークフローが並列ジョブに実行をフォ ークした場合、各ジョブの出力は一貫して表示されます。

ウィンドウ上部にはシェブロンダウンとシェブロンアップのアイコンがあり、ジョブのログ出力を展開したり折りたたんだりすることができます。

### タイムスタンプ

ログ出力は異なるソースからのタイムスタンプを示します：

- **エージェントの時間**：Agentの時間**： *OrderStarted*のような初期のメインイベントはAgentによって作成され、Agentのリアルタイムクロックを反映します。
- **ジョブの時間**：ジョブの出力は、ジョブが実行されるサーバのタイムゾーン、またはジョブの実装から指定されたタイムゾーンを使用します。
- **コントローラの時間**：OrderFinished*などの最終イベントはControllerによって生成され、Controllerのリアルタイムクロックを反映します。

オーダーログビュー(*Order Log View*)では、[Profile - Preferences](/profile-preferences) の設定が有効な場合、タイムスタンプをユーザーのタイムゾーンに変換します。それ以外は、ControllerとAgentのタイムゾーンが使用されます。

コントローラとエージェントのリアルタイムクロックが同期していない場合、タイムトラベルを示唆するログが出力されることがあります。

### ナビゲーション

表示パネルの右側には、ナビゲーションパネルを表示する左矢印のアイコンがあります。

このパネルでは、オーダーによって実行されたジョブやワークフロー指示の履歴を見ることができます。赤色は失敗したジョブやワークフロー指示を示します。

ナビゲーションパネルでジョブをクリックすると、表示パネルの関連ジョブのログ出力にジャンプします。
