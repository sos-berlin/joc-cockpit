# オーダーログ表示

*オーダーログ表示*では、2-3秒ごとに更新される実行中のオーダーログを見ることができます。これにより、オーダーによって実行されたジョブやワークフロー指示の出力をほぼリアルタイムで追うことができます。

## 出力フィルタリング

オーダーログ表示では、ウィンドウ上部から様々な条件によるフィルタリングが可能です：

- **Main** は、オーダー開始、ジョブ開始、ジョブ開始のパラメーターに関する詳細が表示されることを指定します。このような出力は[MAIN]のように示されます。
- **Success** は、[SUCCESS]のように表示され、ジョブが完了したときの結果のパラメータ化などの詳細を表示します。
- **stdout** は、ジョブが標準出力チャネルに書き込んだ出力を表示するかどうかを[STDOUT]のように表示します。
-　**Debug** は、[JS7 - Job API](https://kb.sos-berlin.com/display/JS7/JS7+-+Job+API)を使用するJVMジョブこのようなジョブに、*debug*または*trace*値を保持する*log_level*引数を追加することができます。デバッグ出力がジョブから利用できる場合、[DEBUG]のように表示されます。

ジョブが *標準エラーチャンネル* に書き込んだログ出力はフィルタリングの対象にはならず、[STDERR] のように表示されます。

## ログ出力の表示

ログ出力は到着順に表示されます。ワークフローが並列ジョブに実行をフォ ークした場合、各ジョブの出力は一貫して表示されます。

ウィンドウ上部には下向き矢印と上向矢印のアイコンがあり、ジョブのログ出力を展開したり折りたたんだりすることができます。

### タイムスタンプ

ログ出力は異なるソースからのタイムスタンプを示します：

- **エージェント時間**： *OrderStarted*のような初期のメインイベントはエージェントによって作成され、エージェントのリアルタイムクロックを反映します。
- **ジョブ時間**：ジョブ時間の出力は、ジョブが実行されるサーバのタイムゾーン、またはジョブの実装から指定されたタイムゾーンを使用します。
- **コントローラー時間**：*OrderFinished*などの最終イベントはコントローラーによって生成され、コントローラーのリアルタイムクロックを反映します。

*オーダーログ表示* では、[プロファイル - 設定値](/profile-preferences) の設定が有効な場合、タイムスタンプをユーザーのタイムゾーンに変換します。それ以外は、コントローラーとエージェントのタイムゾーンが使用されます。

コントローラーとエージェントのリアルタイムクロックが同期していない場合、タイムリープを示唆するログが出力されることがあります。

### ナビゲーション

表示パネルの右側には、ナビゲーションパネルを表示する左三角のアイコンがあります。

このパネルでは、オーダーによって実行されたジョブやワークフローオーダーの履歴を見ることができます。赤色は失敗したジョブやワークフロー指示を示します。

ナビゲーションパネルでジョブをクリックすると、表示パネルの関連ジョブのログ出力にジャンプします。
