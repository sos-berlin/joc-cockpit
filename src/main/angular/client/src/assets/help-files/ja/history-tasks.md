# 履歴 - タスク

*履歴 - タスク*画面は、実行されたワークフローやオーダーとは別に、ジョブの実行履歴を表示します。

*履歴 - タスク* は、[クリーンアップサービス](/service-cleanup) によって実行されるデータベース削除の対象となります。

オーダー履歴については、[履歴 - オーダー](/history-orders) を参照してください。

## ナビゲーションパネル

左側のパネルでは、ジョブを実行したワークフローやオーダーからタグによるフィルタリングができます。

- **ワークフロータグ** は、[設定 - インベントリー - ワークフロー](/configuration-inventory-workflows) から設定できます。
- **オーダータグ** は、[設定 - インベントリー - スケジュール](/configuration-inventory-schedules) から設定できます。

タグは + と - のアイコンから選択や、検索アイコンで探すことができます。タグの表示は[設定 - JOC](/settings-joc) から有効にする必要があります。

## 履歴パネル

[プロファイル - 設定値](/profile-preferences) から特に指定がない場合、最大5000件まで表示されます。

### ジョブ履歴

- **ジョブ名** は、関連するジョブ名を表示します。
- **ワークフロー** は、ジョブを実行したワークフロー名を表示します。
  - ワークフロー名をクリックすると [ワークフロー](/workflows) 画面に移動します。
  - 鉛筆アイコンをクリックすると [設定 - インベントリー - ワークフロー](/configuration-inventory-workflows) 画面に移動します。
- **ラベル** は、ワークフロー内でのジョブの位置を示します。ユーザーは表示されるジョブに **ラベル** を設定できます。同じジョブがワークフロー内で複数回発生した場合、異なる **ラベル** が設定できます。
- **ステータス** は、ジョブの結果を示します。
  - ジョブが完了した場合、*ステータス* は *成功* または *失敗* のどちらかになります。
  - ジョブが完了していない場合、*ステータス* は *進行中* となります。

### ログ出力

- **ジョブ名** をクリックすると、そのジョブのログ出力が [タスクログ画面](/task-log) から表示されます。
- **ダウンロード** アイコンをクリックするとタスクログをファイルにダウンロードします。

デフォルトではタスクログの表示は 10MB に制限されています。ユーザーは [設定 - JOC](/settings-joc) 画面から制限を変更できます。

### タスク履歴の操作

タスクごとのアクションメニューでは、以下の操作が可能です：

- **ジョブを無視リストに追加** は、ジョブを永久に非表示にします。これは繰り返し実行され、*タスク履歴* に表示されるジョブに対して有効です。
- **ワークフローを無視リストに追加** は、ワークフローのジョブを永久的に非表示にします。これは、*タスク履歴* に繰り返し入力されるワークフローに便利です。

**無視リスト** は、ウィンドウ右上のボタンから管理します：

- **無視リスト編集** は *無視リスト* 内のジョブやワークフローを表示し、個別に *無視リスト* から削除することができます。 
- **無視リスト有効化**は、*無視リスト* に追加されたジョブ、または追加されたワークフローに含まれるジョブを非表示にするフィルタリングを有効にします。*無視リスト*が有効な場合は、関連するボタンが表示されます。
- **無視リスト無効化** を無効にすると、ジョブやワークフローのフィルタリングが無効になります。この操作はアクティブな *無視リスト* に対して有効です。
- **無視リストをリセット**すると、*無視リスト* からジョブやワークフローが削除され、すべてのジョブが表示されます。

## 詳細検索

ユーザーはウィンドウの上部にある検索ボックスから、ジョブの表示を制限することができます。

- **成功**、**失敗**、**進行中** のボタンは関連する *ステータス* のジョブのみに表示を制限します。
- **日付範囲** は、ジョブ表示の日付範囲を選択します。
- **現在のコントローラー** チェックボックスは、現在選択されているコントローラーのジョブに制限します。
