# ダッシュボード - コントローラーステータス

*コントローラステータス* ウィジェットは、登録されたコントローラーに関する情報を表示します。

## コントローラーインスタンス

表示される各コントローラインスタンスには、以下の項目があります：

- **JS7コントローラーID** は、コントローラーのインストール時に指定された一意の識別子です。コントローラクラースターでは、すべてのインスタンスが同じコントローラーIDを共有します。
- **URL** は、コントローラーにアクセスできるURLを指定します。
- **ステータス** はコンポーネントのステータスを示し、*実行中* またはコントローラーに接続できない場合は *不明* です。
- **クラスターノードステータス** は、クラスター内のコントローラーの *アクティブ* または *スタンバイ* の役割を示します。

コントローラーIDをクリックすると、ダッシュボードが切り替わり、そのコントローラーの製品ステータスが表示されます。

## コントローラーの操作

以下の操作が可能です：

コントローラーは、各インスタンスのアクションメニューから以下の操作が可能です：

- **終了**、**終了後再開** は、インスタンスをシャットダウンします。クラスター内のアクティブなインスタンスでは、メニューが以下に拡張されます：
  - **切替** は、アクティブな役割をスタンバイインスタンスに渡します。
  - **切替さない** は、アクティブな役割をシャットダウンするインスタンスに残します。ユーザーは、フェイルオーバーが行われず、どのインスタンスもアクティブにならないことに注意する必要があります。
- **強制中断**、**強制中断後再開** は、インスタンスを強制的に終了します。クラスター内のアクティブなインスタンスに適用すると、強制的にフェイルオーバーされます：
  - **フェイルオーバーする** は、アクティブな役割をスタンバイインスタンスに渡します。
- **ログダウンロード** コントローラーのcontroller.logファイルをgzip形式の.gzファイルでダウンロードできます。
