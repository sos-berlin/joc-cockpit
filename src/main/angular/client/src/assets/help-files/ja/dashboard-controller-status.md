# コントローラーステータス

コントローラステータス*パネルは、登録されたコントローラに関する情報を提供します。

## コントローラインスタンス

表示される各コントローラインスタンスには、以下の属性があります：

- **コントローラーID**は、コントローラーのインストール時に指定された一意の識別子です。コントローラクラスタでは、すべてのインスタンスが同じコントローラIDを共有します。
- **URL**は、ControllerインスタンスにアクセスできるURLを指定します。
- **Status**はコンポーネントのステータスを示し、*operational*またはControllerインスタンスに到達できない場合は*unknown*です。
- **クラスタの役割** は、クラスタ内のControllerインスタンスの*アクティブ*または*スタンバイ*の役割を示します。

インスタンスのController IDをクリックすると、ダッシュボードが切り替わり、関連するControllerの製品ステータスが表示されます。

## コントローラインスタンスの操作

以下の操作が可能です：

コントローラーインスタンスでは、各インスタンスの3ドットアクションメニューから以下の操作が可能です：

- **Terminate**、**Terminate and Restart**はインスタンスをシャットダウンします。クラスタ内のアクティブなインスタンスでは、メニューが拡張されます：
  - **switch-over** でアクティブな役割をスタンバイ・インスタンスに渡します。
  - **スイッチ・オーバーなし**：アクティブな役割をシャットダウンされたインスタンスに残します。ユーザーは、フェイルオーバーが行われず、どのインスタンスもアクティブにならないことに注意する必要があります。
- **Cancel**、**Cancel and Restart**はインスタンスを強制的に終了します。クラスタ内のアクティブなインスタンスに適用すると、強制的にフェイルオーバーされます：
  - **with fail-over**はアクティブな役割をスタンバイ・インスタンスに渡します。
- **Download Log** コントローラのcontroller.logファイルをgzip形式の.gzファイルからダウンロードできます。
