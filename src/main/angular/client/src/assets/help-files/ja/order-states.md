# オーダーソース

オーダーは以下のソースから追加することができます：

- [実行計画](/daily-plan)によって自動作成されたオーダー。
- [ワークフロー](/workflows) 画面からユーザーが追加したオーダー。
- ファイルオーダーソースから追加されたオーダー。

## オーダーステータス

以下の *オーダーステータス* が利用可能です：

- **保留** は開始時刻を指定しないオーダー。
- **予定** オーダーは登録され、後日実行される予定です。
- **進行中** オーダーはワークフロー指示によって実行されますが、ジョブは実行されていません。 
- **実行中** オーダーはジョブ実行中です。 
- **一時停止** オーダーはユーザーの介入により中断され、再開することができます。
- **完了** オーダーはワークフローを完了しましたが、削除されていません。例えば、ファイルオーダーソースがファイル監視に使用され、ワークフローが受信ファイルを（再）移動しない場合などです。この場合、ファイルが受信ディレクトリに存在する限り、オーダーはそのまま残ります。
- **指示待ち** オーダーは、ワークフローの *プロンプト命令* により保留され、ワークフローの実行を続行するためにユーザーの確認が必要です。
- **待機中** オーダーは、*ロック*、*Notice*、*Retry*、または *Cycle* インターバルなどによって待機します。
- **ブロック中** オーダーは、オーダーが追加されてからエージェントに到達できない場合などに開始できません。
- **失敗** オーダーは、ジョブが失敗したか、*Fail命令*によってオーダーが進行できないことを示します。 

表示された数のオーダーをクリックすると、[オーダー表示](/orders-overview) に移動し、オーダーの詳細が表示されます。
