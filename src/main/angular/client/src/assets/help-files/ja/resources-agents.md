# エージェントのステータス

Agent*ビューは、登録されたエージェントの接続ステータスを要約します。

## アーキテクチャ

### エージェント

- **スタンダロンエージェント**は、オンプレミスとコンテナからリモートマシン上でジョブを実行します。エージェントは個別に運用され、コントローラによって管理されます。
- **エージェントクラスタ
  - **Directorエージェント**は、エージェントクラスタ内の*Subagents*をオーケストレーションします。これらのエージェントはアクティブパッシブクラスタリングで2つのインスタンスから運用され、Controllerによって管理されます。
  - **Subagents**は、オンプレミスのリモートマシンやコンテナからジョブを実行します。これらはエージェントクラスタのワーカーノードと考えることができ、**Director Agent*によって管理されます。

### 接続

- **Standalone Agent**, **Director Agent** 接続は、Controller によって確立されます。 
- エージェントクラスタ内の**サブエージェント**接続は、**Director Agent**によって確立されます。

## 接続ステータス

エージェントのステータス表示には、以下のカラーインジケータが使用されます：

- **緑色** は健全なエージェント接続を示します。
- **黄色は現在リセット中のエージェントを示します。
- **赤色は、エージェントに到達できない場合など、エージェントへの接続が失敗していることを示します。
- **グレー色** は、接続ステータスが *不明* であることを示します。例えば、Director エージェントに接続できない場合、Subagent のステータスは *不明* です。

ユーザは以下の影響を考慮する必要があります：

- エージェント接続が失敗とみなされた場合、エージェントがダウンしていることを示すものではありません。接続を妨げるネットワークの問題がある可能性があります。
- JOC コックピットは、コントローラからエージェント接続ステータスの情報を受け取ります。コントローラが利用できない場合、この情報は表示されません。これはエージェントがダウンしていることを意味しませんが、エージェントは *不明* ステータスから表示されることを意味します。
- コントローラーは、*Standalone Agent* と *Director Agent* への接続を報告します。Director Agent* への接続に失敗した場合、Controller はエージェントクラスタ内の *Subagents* のステータスを把握していません。
