# リソース - エージェント

*エージェント* 画面は、登録されたエージェントのステータスを表示します。

## アーキテクチャー

### エージェント

- **スタンドアロンエージェント** は、オンプレミス環境のコンテナやリモートマシン上でジョブを実行します。エージェントは個別に運用され、コントローラーによって管理されます。
- **エージェントクラスター**
 - **ディレクターエージェント** は、エージェントクラスター内の *サブエージェント* を管理します。ディレクターエージェントはアクティブスタンバイ構成での2つのインスタンスから構成可能で、コントローラーによって管理されます。
 - **サブエージェント** は、オンプレミス環境のリモートマシンやコンテナからジョブを実行します。これらはエージェントクラスターのワーカーノードと考えることができ、**ディレクターエージェント** によって管理されます。

### 接続

- **スタンドアロン**、**ディレクターエージェント** へは、コントローラーから接続されます。
- エージェントクラスター内の **サブエージェント** へは、**ディレクターエージェント** から接続されます。

## ステータス

エージェントのステータス表示には、以下のカラーインジケータが使用されます：

- **緑** は、健全なエージェント接続を示します。
- **黄** は、現在リセット中のエージェントを示します。
- **赤** は、エージェントに到達できない場合など、エージェントへの接続が失敗していることを示します。
- **グレー** は、接続ステータスが *不明* であることを示します。例えば、ディレクターエージェントに接続できない場合、サブエージェントのステータスは *不明* になります。

ユーザは以下の影響を考慮する必要があります：

- エージェント接続が失敗とみなされた場合、エージェントがダウンしていることを示すものではありません。接続を妨げるネットワークの問題がある可能性があります。
- JOCコックピットは、コントローラーからエージェント接続ステータスの情報を受け取ります。コントローラーが利用できない場合、この情報は表示されません。これはエージェントがダウンしていることを意味しませんが、エージェントは *不明* ステータスから表示されることを意味します。
- コントローラーは、*スタンドアロン* と *ディレクターエージェント* への接続を報告します。*ディレクターエージェント* への接続に失敗した場合、コントローラーはエージェントクラスター内の *サブエージェント* のステータスを把握できていません。
