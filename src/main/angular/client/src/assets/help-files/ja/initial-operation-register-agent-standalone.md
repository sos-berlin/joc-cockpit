# 初期設定 - スタンドアロンエージェント登録

初期設定は、コントローラー、エージェント、JOCコックピットのインストール後に実行します。エージェント登録は[初期設定 - コントローラー登録](/initial-operation-register-controller) 完了後に行います。

## スタンドアロンエージェントの登録

コントローラーのサーバーからエージェントのサーバーへのネットワーク接続が可能であること、ファイアウォールのルールがエージェントのポートへの接続を許可していることを確認してください。

メインメニューバーのホイールアイコンから *コントローラー/エージェント管理* 画面を開き、コントローラーのアクションメニューから *単独エージェント追加* を選択します。これにより、スタンドアロンエージェントの登録ポップアップウィンドウが表示されます。

ユーザーは以下の入力を行います：

- **エージェントID** は、エージェントの一意の識別子で、エージェントの有効期間中は変更できません。*エージェントID* は、ジョブおよびワークフローでは表示されません。
  - FQDN+ポート番号など一意なものを使用します
- **エージェント名** は、エージェントの一意の名称です。ジョブにエージェントを割り当てる場合、*エージェント名* を使用します。
   - 本番環境とテスト環境で同じ *エージェント名* を使用することができますので、例えば *会計*、*販売* など業務上の名称が使用できます。
   - *エージェント名* を後で変更する場合は、*エイリアス名* で以前の *エージェント名* を引き続き使用できます。
- **タイトル** は、エージェントに追加できる説明です。
- **エイリアス名** は、同じエージェントの別名です。ジョブにエージェントを割り当てる場合、*エイリアス名* も使用できます。 *エイリアス名* は、例えばテスト環境が本番環境より少ないエージェントを含んでいる場合にも使用でき、この場合、環境間でエージェントの割り当てを変更せずに維持するため、不足しているエージェントは同じエージェントのエイリアス名にマッピングされます。
- **URL** は、エージェントへの接続にコントローラーが使用するプロトコル、ホスト、ポート番号、例えば http://localhost:4445 から URL を指定します。
  - エージェントが HTTP を使用する場合、URL は *http* から始まります。エージェントが HTTPS 用に設定されている場合は、*https* から始まります。
  - エージェントがコントローラーと同じマシンにインストールされている場合、ホスト名は *localhost* になります。それ以外の場合は、エージェントのホストの FQDN を指定する必要があります。
  - エージェントの *ポート番号* はインストール時に決定されます。 

登録に成功すると、エージェントは[リソース - エージェント](/resources-agents) 画面に表示されます。
