# リソースロック

リソースロック*ビューには、リソースロックの使用に関するライブ情報が表示されます。

リソースロックは、ワークフロー間でジョブやワークフロー命令の並列性を制限するために使用されます。
リソースロックは、同じワークフロー内の複数のジョブやワークフロー指示にまたがるブロック命令です。

- **排他ロック**は、1つのジョブで使用することができます。排他アクセスは、リソースロックまたはワークフローでの使用から設定されます。
- **共有ロック**は設定可能な数のジョブが使用できます。
  - リソースロックには*容量*が割り当てられます。
  - 例えば、ワークフローAとワークフローBで使用する場合、3と4です。これにより、ワークフローAの2つのオーダーの並列実行が可能になり、ワークフローAとワークフローBのオーダーの並列実行は拒否されます。

## ナビゲーションパネル

左側のパネルにはリソースロックを保持するインベントリフォルダのツリーが表示 されます。

- フォルダをクリックすると、そのフォルダのリソースロックが表示されます。
- フォルダをクリックすると、そのフォルダのリソースロックが表示されます。

クイック検索アイコンをクリックすると、ユーザー入力に基づいてリソースロックを検索することができます：

- Test**と入力すると、*test-lock-1**や*test-lock-2**といった名前のリソースロックが表示されます。 
- Test***と入力すると、*test-lock-1**や*my-TEST-lock-2**といった名前のリソースロックが表示されます。

## リソースロックパネル

### リソースロックの表示

以下の情報が表示されます：

- **Name** はリソース・ロックの一意の名前です。
- **Deployment Date** はリソース・ロックがデプロイされた日付です。
- **Status**は、リソースロックがコントローラにデプロイされていない場合、*Synchronized*と*Not Synchronized*のいずれかです。
- 獲得ウェイト**は、ロックを獲得した並列オーダーによる累積ウェイト**を示します。
- **Holding Orders** は、ロックを取得したオーダーの数を示します。
- **待機中のオーダー** は、ロックの取得を待機しているオーダーの数を示します。
- キャパシティ** は、そのロックで利用可能な*キャパシティ** を示します。 *排他的ロック*は1つの*容量を保持し、*共有ロック*は個々の*容量を保持します。

### オーダーの表示

矢印アイコンをクリックすると、リソースロックが展開され、リソースロックを取得したオー ダーやリソースロック待ちのオーダーの詳細情報が表示されます。

## 検索

例えば、特定のジョブ名を含むワークフローを検索すると、そのワークフローで使用されているリソースロックが返されます。
