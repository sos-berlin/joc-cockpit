# リソース - ロック

ロック画面には、ロックの使用状態が表示されます。

ロックは、ワークフロー間でジョブやワークフロー命令の並列性を制御するために使用されます。
ロックは、同じワークフロー内の複数のジョブやワークフロー指示にまたがるブロック命令です。

- **排他ロック**は、1つのジョブで使用できます。排他アクセスは、ロックまたはワークフローから設定できます。
- **共有ロック**は設定可能な数のジョブで使用できます。
  - 共有ロックには*取得可能ロック数*が割り当てられます。例：*6*
  - 共有ロックをジョブセットが使用する場合、各使用に対して重み付けが割り当てられます（例：ワークフローAとBでそれぞれ3と4）。この仕組みにより、ワークフローAのオーダー2件を並列実行できる一方、ワークフローAとBの注文を同時に実行することは禁止されます。これにより、ワークフローAの2つのオーダーの並列実行が可能になり、ワークフローAとワークフローBのオーダーの並列実行は拒否されます。

## ナビゲーションパネル

左側のパネルにはロックを保持するインベントリーフォルダのツリーが表示 されます。

- フォルダーをクリックすると、そのフォルダーのロックが表示されます。
- フォルダーにマウスカーソルを合わせると表示される三角アイコンをクリックすると、そのフォルダーおよびすべてのサブフォルダーに設定されているロックが表示されます。

クイック検索アイコンをクリックすると、ユーザー入力に基づいてロックを検索することができます：

- **Test**と入力すると、*test-lock-1* や *test-lock-2* といった名前のロックが表示されます。 
- **\*Test**と入力すると、*test-lock-1* や *my-TEST-lock-2* といった名前のロックが表示されます。

## ロック画面

### ロックの表示

以下の情報が表示されます：

- **名称** はロックの一意の名前です。
- **配置日時** はロックが配置れた日付です。
- **ステータス**は、ロックがコントローラーに配置されていない場合、*同期済* か *未同期* のいずれかです。
- **取得ロック数**は、ロックを獲得した並列オーダーによる *累積ロック数* を示します。
- **ロック保持オーダー** は、ロックを取得したオーダーの数を示します。
- **ロック待機オーダー** は、ロックの取得を待機しているオーダーの数を示します。
- **制限**は、そのロックで利用可能な *ロック数* を示します。 *排他的ロック*は1個の *ロック数* であり、*共有ロック* は個々の*ロック数* 容量を保持します。

### オーダーの表示

三角アイコンをクリックすると、ロックが展開され、ロックを取得したオーダーやロック待ちのオーダーの詳細情報が表示されます。

## 検索

例えば、特定のジョブ名を含むワークフローを検索すると、そのワークフローで使用されているロックが返されます。
