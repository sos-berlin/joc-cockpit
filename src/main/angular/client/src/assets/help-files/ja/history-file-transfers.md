# 履歴 - ファイル転送

*履歴 - ファイル転送* 画面は、YADE ファイル転送ジョブのオーダーの実行履歴をまとめたものです。

ファイル転送ジョブから作成されたログ出力については[履歴 - タスク](/history-tasks) を参照してください。オーダーの履歴については[履歴　ーオーダー](/history-orders) を参照してください。

## 履歴パネル

### ファイル転送履歴

ファイル転送操作のブロックと個別ファイルの転送ブロックに分かれて表示されます：

- **ステータス** は、転送が *成功* または *失敗* したかを示します。
  - *成功* は、転送のすべてのファイルが正常に処理されたことを示します。
  - *失敗* は、転送内の1つ以上のファイルがエラーで処理されたことを示します。
- **プロファイル名** は、ファイル転送プロファイルの一意の識別子です。
- **操作** は、*コピー*、*移動*、*削除*、*一覧取得* のいずれかを指定します。
- **ワークフロー** は、オーダーによって実行されるワークフローを示します。
  - ワークフロー名をクリックすると、[ワークフロー](/workflows) 画面0に移動します。
  - 鉛筆アイコンをクリックすると、[設定 - インベントリー - ワークフロー](/configuration-inventory-workflows) 画面に移動します。
- **オーダーID** は、オーダーに割り当てられた一意の識別子です。
- **合計** は、転送に含まれるファイル数を示します。

### ファイルごとの履歴

ファイル転送操作には、任意の数のファイルを含めることができます。*ファイル転送履歴*は、転送から利用可能な矢印ダウンアイコンをクリックすると、ファイルごとの転送ステータスを表示します：

表示される情報は、以下のブロックに分類されます：

- **ソースホスト** は、転送元を示します。
- **ターゲットホスト** は、転送先を示します。
- **ジャンプホスト** は、ソースとターゲット間の中継ホストの使用を示します。*ジャンプ* ホストは、ソースとターゲットの間でファイル転送を直接実行できないが、送受信操作のためにDMZ内のホストが必要な場合に使用されます。

*ソースホスト*、*ターゲットホスト*、*ジャンプホスト*の詳細が表示されます：

- **ホスト** は、サーバーのホスト名またはIPアドレスを示します。
- **アカウント** は、サーバーへのアクセスに使用するユーザーアカウントを示します。
- **ポート** は、サーバーへの接続に使用するポート番号を示します。
- **プロトコル** は、FTP、FTPS、SFTP、CIFSなどのファイル転送プロトコルを示します。

*ソースホスト*と*ターゲットホスト*については、以下の詳細が表示されます：

- **ファイル名** は、ファイル名を示します。
- **ファイルパス** は、ファイルへのディレクトリパスとファイル名を示します。
- **ステータス**
  - **転送済** は、*コピー* または *移動* 操作でファイルが正常に転送されたことを示します。
  - **削除済** は、*削除* 操作でファイルが削除されたことを示します。
  - **スキップ** は、*一覧取得* 操作などでファイルが転送から除外されたことを示します。
- **サイズ** は、転送されたバイト数を指定します。
- **ハッシュ値** は、転送に使用された MD5 ハッシュを示します。
