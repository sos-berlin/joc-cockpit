# ファイル転送履歴

ファイル転送履歴* ビューは、YADE マネージドファイル転送ジョブのオーダーの実行履歴をまとめたものです。

ファイル転送ジョブから作成されたログ出力については[Task History](/history-tasks) を参照してください。オーダーの履歴については[Order History](/history-orders) を参照してください。

## 履歴パネル

### 転送履歴

転送操作のブロックと個別ファイルの転送ブロックに分かれて表示されます：

- **履歴ステータス**は、転送が*成功*または*失敗*したかを示します。
  - *成功*は、転送のすべてのファイルが正常に処理されたことを示します。
  - 失敗**は、転送内の1つ以上のファイルがエラーで処理されたことを示します。
- **プロファイル名**は、ファイル転送プロファイルの一意の識別子です。
- **操作**は、*COPY*、*MOVE*、*REMOVE*、*GETLIST*のいずれかを指定します。
- **ワークフロー**は、オーダーによって実行されるワークフローを示します。
  - ワークフロー名をクリックすると、[Workflows](/workflows) ビューに移動します。
  - 鉛筆アイコンをクリックすると、[Configuration - Inventory - Workflows](/configuration-inventory-workflows) ビューに移動します。
- **オーダーID**はオーダーに割り当てられた一意の識別子です。
- **合計**は、転送に含まれるファイル数を示します。

### ファイルごとの履歴

ファイル転送操作には、任意の数のファイルを含めることができます。ファイル転送履歴*は、転送から利用可能な矢印ダウンアイコンをクリックすると、ファイルごとの転送ステータスを表示します：

表示される情報は、以下のブロックに分類されます：

- **ソース**は転送元を示します。
- **転送元**は、転送元を示します。
- **Jump**は、ソースとターゲット間のジャンプ・ホストの使用を示します。Jump*ホストは、ソースとターゲットの間でファイル転送を直接実行できないが、送受信操作のためにDMZ内のホストが必要な場合に使用されます。

Source*、*Target*、*Jump* ホストの詳細が表示されます：

- **ホスト**はサーバーのホスト名またはIPアドレスを示します。
- **ホスト**は、サーバーのホスト名または IP アドレスを示します。
- **Port**は、サーバーへの接続に使用するポートを示します。
- **プロトコル**は、FTP、FTPS、SFTP、CIFSなどのファイル転送プロトコルを示します。

Source*と*Target*については、以下の詳細が表示されます：

- **ファイル名**はファイル名を示します。
- **ファイル名**はファイルの名前を示します。
- **ステータス
  - **TRANSFERRED**は、*COPY*または*MOVE*操作でファイルが正常に転送されたことを示します。
  - **DELETED**は、*REMOVE*操作でファイルが削除されたことを示します。
  - **SKIPPED**は、*GETLIST*操作などでファイルが転送から除外されたことを示します。
- **Size**は、転送されたバイト数を指定します。
- 関連オプションが転送に使用された場合、**Integrity Hash** は MD5 ハッシュを示します。
