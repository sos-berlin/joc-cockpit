# IDサービス管理 - プロファイル

IDサービスは、認証と認可によってJOCコックピットへのアクセスを管理します。[IDサービス管理](/identity-services) を参照してください。

IDサービスでログインするユーザーは、初回ログイン時に関連する IDサービスの *プロファイル* が作成されます。

- 複数の IDサービスでログインできる場合、ユーザーはログインに成功したそのIDサービスで *プロファイル* を持つことになります。
- *プロファイル*は、[設定 - JOC](/settings-joc) 画面の *default_profile_account* で指定されたアカウントから作成されます。デフォルトでは、*root* アカウントの *プロファイル* が使用されます。
- 長期間使用されない場合、*プロファイル* は削除されます。[設定 - Cleanup](/settings-cleanup) 画面の *profile_age* で、ユーザーアカウントからログインが発生しない場合 *プロファイル* が保持される最大期間を指定します。

## プロファイルの操作

プロファイル画面では、有効な *プロファイル* のリストと最終ログイン日が表示されます。*プロファイル* に対して個別に以下の操作が可能です：

-  *プロファイル* をクリックすると、[IDサービス管理 - ロール管理](/identity-service-roles) 画面に移動し、指定された *プロファイル* で使用されているロールが表示されます。
- *プロファイル* のアクションメニューには以下の操作があります：
  - **プロファイルプリファレンスを削除** は、[プロファイル - プリファレンス](/profile-preferences) をデフォルトにリセットします。*Git管理* や *お気に入り管理* などの他の *プロファイル* 設定はそのまま残ります。この操作は、デフォルトアカウントの *プロファイル* の適用を強制するために使用できます。
  - **プロファイルと変更を削除**は、ユーザーアカウントの *プロファイル* を削除します。関連アカウントの次回ログイン時に、新しい *プロファイル* が作成されます。

ユーザーは 1 つ以上の *プロファイル* を選択して、選択した *プロファイル* に対して上記の操作を一括して実行できます。
