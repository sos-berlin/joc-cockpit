# 設定 - インベントリ - 操作 - フォルダのエクスポート

オブジェクトのエクスポートには、ダウンロード用に提供される .zip または .tar.gz アーカイブファイルへの追加も含まれます。これは、*Controller* および *Automation* システムフォルダのオブジェクトと、ユーザーフォルダのオブジェクトに適用されます。アーカイブファイルは、同じJOC Cockpitインスタンスまたは別のJOC Cockpitインスタンスに後でインポートするために使用できます。

フォルダの3点アクションメニューから関連する*Export*操作を使用してフォルダからオブジェクトをエクスポートすると、ポップアップウィンドウが表示されます。 

- **ファイル名**はアーカイブファイルの名前を指定します。
- **ファイル形式**は、圧縮タイプに.zipまたは.tar.gzを指定します。
- **エクスポート・タイプ(Export Type)]は次のいずれかです。
  - 個々のオブジェクトのエクスポート
  - フォルダのエクスポート
  - 変更のエクスポート
- **Filer Type**
  - **Controller**は、*Controller*システムフォルダに格納されたワークフローやジョブリソースなどのオブジェクトを考慮します。
  - **Automation**システムフォルダに保存されたスケジュールやカレンダーなどのオブジェクトを考慮します。
- **フィルタ
  - **有効なオブジェクトにエクスポートを制限します。
  - **下書き**には、下書きステータスのオブジェクトが含まれます。
  - **デプロイ済み**には、デプロイ済みステータスのワークフローやジョブリソースなどのオブジェクトが含まれます。
  - **リリース済み**には、リリースされたステータスのスケジュールやカレンダーなどのオブジェクトが含まれます。 
  - **相対パスを使用する]は、エクスポートファイルが絶対パスからフォルダ階層を含むか、またはエクスポートが実行される階層内の最新のフォルダによって示される相対パスからフォルダ階層を含むかを指定します。
- **Handle recursively** **サブフォルダからエクスポート・アーカイブに再帰的にオブジェクトを追加できます。

## エクスポートの種類

エクスポートの種類**では、個々のオブジェクト、フォルダからのオブジェクト、変更点からのオブジェクトを選択できます。

### 個々のオブジェクトのエクスポート

エクスポートタイプ**では、表示されているオブジェクトリストから個々のオブジェクトを選択できます。

<img src="export-object.png" alt="Export Object" width="600" height="580" />

### フォルダからのオブジェクトのエクスポート

エクスポートタイプ(*Export Type*)は、ワークフローやスケジュールのような特定のスケジューリングオブジェクトタイプにエクスポートを制限します。ユーザーは、エクスポートアーカイブファイルに追加するオブジェクトタイプを選択できます。

<img src="export-folder.png" alt="Export Folder" width="600" height="580" />

### 変更からオブジェクトをエクスポート

エクスポートタイプ*では、[Changes](/changes) のリストから変更を選択できます。 エクスポートは、変更に関連するオブジェクトに限定されます。

## 依存関係を含む

インベントリオブジェクトは依存関係によって関連付けられます。[Dependency Matrix](/dependencies-matrix) を参照してください。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトをエクスポートする際、整合性が考慮されます：

- 例えば、ワークフローがジョブリソースを参照している場合、ワークフローとジョブリソースは、選択されたフォルダとは無関係のフォルダに保存されていてもエクスポートすることができます。
- スケジュールがカレンダーを参照している場合、スケジュールとカレンダーの両方をエクスポートすることができます。

ユーザーは以下のオプションから一貫性のあるエクスポートをコントロールします：

- **Include Dependencies**
  - をチェックすると、任意のフォルダにある参照オブジェクトと被参照オブジェクトの両方が含まれます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、デプロイ/リリースされているかどうかを確認する必要があります。一貫性のないデプロイによってオブジェクトが見つからない場合、 コントローラはエラーメッセージを表示します。
