# ジョブ定義 - インベントリー - 操作 - エキスポート

オブジェクトのエキスポートには、ダウンロード用に .zip または .tar.gz アーカイブファイルへの追加も含まれます。これは、*コントローラー* および *実行計画* システムフォルダーのオブジェクトと、ユーザーフォルダーのオブジェクトに適用されます。アーカイブファイルは、同じJOCコックピットまたは別のJOCコックピットに後でインポートするために使用できます。

フォルダーのアクションメニューから関連する *エキスポート* ボタンから、ポップアップウィンドウが表示されます。 

- **ファイル名** は、アーカイブファイルの名前を指定します。
- **ファイル形式** は、圧縮タイプに.zipまたは.tar.gzを指定します。
- **エキスポートタイプ** は、次のいずれかです。
  - **個別のオブジェクト**
  - **フォルダー**
  - **変更**
- **フィルタータイプ**
  - **コントローラー** は、*コントローラー* システムフォルダーに保存されているワークフローやジョブリソースなどのオブジェクトをエキスポートします。
  - **実行計画** は、*実行計画* システムフォルダーに保存されているスケジュールやカレンダーなどのオブジェクトをエキスポートします。
- **フィルター**
  - **有効のみ** は、有効なオブジェクトのみエキスポートします。
  - **下書き** は、下書きステータスのオブジェクトが含まれます。
  - **配置済** には、配置済ステータスのワークフローやジョブリソースなどのオブジェクトが含まれます。
  - **リリース済** には、リリース済ステータスのスケジュールやカレンダーなどのオブジェクトが含まれます。 
  - **相対パスを使用する** は、エキスポートファイルが絶対パスからフォルダ階層を含むか、またはエキスポートが実行される階層内の最新のフォルダによって示される相対パスからフォルダ階層を含むかを指定します。
- **フォルダー内にも適用** は、サブフォルダー内のオブジェクトも再帰的に追加できます。

## エキスポートの種類

**エキスポートの種類** では、個々のオブジェクト、フォルダーからのオブジェクト、変更点からのオブジェクトを選択できます。

### 個別のオブジェクトのエキスポート

**エキスポートタイプ**では、表示されているオブジェクトリストから個々のオブジェクトを選択できます。

### フォルダーからのオブジェクトのエキスポート

*エキスポートタイプ* は、ワークフローやスケジュールのような特定のオブジェクトタイプにエキスポートを制限します。ユーザーは、エキスポートファイルに追加するオブジェクトタイプを選択できます。

<img src="export-folder.png" alt="Export Folder" width="600" height="580" />

### 変更をエキスポート

*エキスポートタイプ* [変更](/changes) のリストからオブジェクトを選択できます。 エキスポートは、変更に関連するオブジェクトに限定されます。

## 依存関係を含む

インベントリーオブジェクトは依存関係によって関連付けられます。[依存関係](/dependencies-matrix) を参照してください。例えば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つ以上のワークフローを参照します。

オブジェクトをエキスポートする際、整合性が考慮されます：

- 例えば、ワークフローがジョブリソースを参照している場合、ワークフローとジョブリソースは、選択されたフォルダとは無関係のフォルダに保存されていてもエキスポートすることができます。
- スケジュールがカレンダーを参照している場合、スケジュールとカレンダーの両方をエキスポートすることができます。

ユーザーは以下のオプションから一貫性のあるエクスポートをコントロールします：

- **依存関係を含む** をチェックすると、任意のフォルダにある参照オブジェクトと被参照オブジェクトの両方が含まれます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、配置/リリースされているかどうかを確認する必要があります。一貫性のない配置によってオブジェクトが見つからない場合、 コントローラーはエラーメッセージを表示します。
