# 設定 - インベントリ - 操作 - フォルダの展開

オブジェクトのデプロイには、オブジェクトを1つまたは複数のコントローラに転送することが含まれます。これは *Controller* システムフォルダから利用可能なワークフローやジョブリソースなどのオブジェクトに適用されます。

Configuration-&gt;Inventory* ビューでは、単一のオブジェクトのデプロイ（[Configuration - Inventory - Operations - Deploy Object](/configuration-inventory-operations-deploy-object) を参照）、およびフォルダからのオブジェクトのデプロイが可能です。

フォルダの 3-dots アクションメニューから関連する *Deploy* 操作を使用してフォルダからオブジェクトをデプロイすると、次のようなポップアップウィンドウが表示されます：

<img src="deploy-folder.png" alt="Deploy Folder" width="600" height="460" />

## コントローラーへのデプロイ

入力フィールド **Controller** には、オブジェクトをデプロイするコントローラを指定します。

デフォルトでは、現在選択されているコントローラが使用されます。

## 実行計画の更新

ワークフロー、スケジュール、カレンダーなどのオブジェクトの展開は、[Daily Plan](/daily-plan) に影響を与えます。頻繁に、オブジェクトの更新されたバージョンが実行計画のオーダーに使用されます。ユーザーは以下のオプションから動作を制御します：

- **実行計画を更新します。
  - **Now**は、現在から開始する時点でスケジュールされているオーダーの実行計画を更新します。
  - **Start Date** を選択すると、実行計画を更新する開始日の入力フィールドが追加されます。
  - **いいえ] を選択すると、実行計画は更新されません。既存のオーダーは、以前にデプロイされたバージョンのオブジェクトを使用します。
- **今日の遅れたオーダーを含める]**にチェックを入れると、現在日中の過去の時間にスケジュールされ、遅れて開始されなかったオーダーが含まれます。

## オブジェクトと変更のデプロイ

デプロイタイプ]では、以下から選択できます。 

- **選択されたフォルダで利用可能な個々のオブジェクト**。
- [Change](/changes) の対象となるオブジェクトを含む **変更** . ユーザーはリストボックスから利用可能な変更のリストから目的の変更を選択します。

## オブジェクトのフィルタリング

オブジェクトは以下のオプションからフィルタリングできます：

- **Draft**は、ドラフト状態のオブジェクトをデプロイすることを指定します。
- **Deployed**は、デプロイ状態のオブジェクトをデプロイに含めることを指定します。

## サブフォルダを含む

Handle recursively** オプションを使用すると、サブフォルダのオブジェクトを配備に 再帰的に追加できます。

## 依存関係を含める

インベントリオブジェクトは依存関係によって関連付けられます（[Dependency Matrix](/dependencies-matrix) を参照）。たとえば、ワークフローはジョブリソースとリソースロックを参照し、スケジュールはカレンダーと1つまたは複数のワークフローを参照します。

オブジェクトをデプロイする際には、整合性が考慮されます：

- ジョブリソースが作成され、新しく作成されたワークフローによって参照される場合、ワークフローのデプロイはジョブリソースのデプロイを含みます。
- 例えば、ジョブリソースが作成され、新しく作成されたワークフローによって参照される場合、ワークフローのデプロイにはジョブリソースのデプロイも含まれます。ジョブリソースがデプロイされたワークフローによって参照され、破棄または削除される場合、ワークフローも破棄または削除されなければなりません。

ユーザーは以下のオプションから一貫性のあるデプロイメントをコントロールします：

- **Include Dependencies**
  - をチェックすると、参照オブジェクトと被参照オブジェクトの両方が含まれます。
    - 関連するオブジェクトがドラフト ステータスの場合、共通のデプロイメントが提供されます。このオプションは、オブジェクトの関係の変更によって必要な場合、強制されます。
    - 関連オブジェクトが配置済み/リリース ステータスの場合、共通配置はオプションです。ユーザーは、関連オブジェクトを選択して共通ディプロイメントを行うことができます。
  - チェックを外すと、依存関係は考慮されません。ユーザーは、関連オブジェクトが有効で、デプロイ/リリースされているかどうかを確認する必要があります。一貫性のないデプロイによってオブジェクトが見つからない場合、コントローラはエラーメッセージを表示します。
