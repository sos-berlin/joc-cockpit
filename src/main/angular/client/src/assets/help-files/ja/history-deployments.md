# 配備履歴

デプロイ履歴* ビューは、インベントリオブジェクトのデプロイを要約します。

JS7 は、異なるエージェントやプラットフォーム上で同じワークフローのジョブを実行できる分散アーキテクチャを実装しています。デプロイが成功すると、各エージェントから非同期でオブジェクトの新規・更新が確認されます。

Configuration* ビューで *deploy* オペレーションを使用する場合、Agent の可用性に応じて確認が行われます。たとえば、シャットダウンしている Agent は、再起動時にデプロイを確認します。

デプロイ履歴* は、インベントリオブジェクトのデプロイ状況を反映するために非同期に更新されます。

## 履歴パネル

### デプロイの履歴

表示は、デプロイメントごとに 1 つのブロックにまとめられ、インベントリオブジェクト ごとにブロックに分けられています。

- **デプロイメント日付**は、デプロイメントの時点を示します。
- **アカウント**は、配置操作を実行した JOC Cockpit ユーザーアカウントを示します。
- **配置が成功したか失敗したかを示します。
  - *デプロイ済み]は、すべてのインベントリ・オブジェクトが正常にデプロイされたことを示します。
  - *デプロイされませんでした]は、1 つ以上のインベントリ・オブジェクトがデプロイされなかったことを示します。
- **アイテム数** は、ワークフロー、ジョブリソースなどのインベントリオブジェクトの数を示します。

### インベントリオブジェクトごとのデプロイ履歴

インベントリオブジェクトごとの詳細が表示されます：

- **メッセージ** は、デプロイに失敗した場合のエラーメッセージを示します。
- **オブジェクトタイプ** は、*ワークフロー*、*ジョブリソース* などのインベントリオブジェクトのタイプを示します。
- **パス** はインベントリ・フォルダとオブジェクト名を示します。ワークフローオブジェクトの場合
  - ワークフロー名をクリックすると[Workflows](/workflows) ビューに移動します、
  - 鉛筆アイコンをクリックすると[Configuration - Inventory - Workflows](/configuration-inventory-workflows) ビューに移動します。
- **操作** は、*store* または *delete* のいずれかです。更新されたオブジェクトは両方の操作で発生します。
- **日付**は、デプロイ操作の時点です。
