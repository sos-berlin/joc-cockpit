# 履歴 - 配置

*履歴 - 配置* 画面は、インベントリーオブジェクトの配置状況を表示します。

JS7 は、異なるエージェントやプラットフォーム上で同じワークフローのジョブを実行できる分散アーキテクチャーを実装しています。配置が成功すると、各エージェントから非同期でオブジェクトの新規・更新が確認されます。

*設定* 画面から *配置* 操作をする場合、エージェントの状態に応じて確認が行われます。たとえば、停止しているエージェントには、再起動時に配置を確認します。

*履歴 - 配置* は、インベントリーオブジェクトの配置状況を反映するために非同期に更新されます。

## 履歴パネル

### 配置履歴

表示は、配置ごとに 1 つのブロックにまとめられ、インベントリーオブジェクトごとにブロックに分けられています。

- **配置日時** は、配置された日時を示します。
- **アカウント** は、配置を実行した JOCコックピットのユーザーアカウントを示します。
- **ステータス** は、配置が成功したか失敗したかを示します。
  - *配置* は、すべてのインベントリーオブジェクトが正常に配置されたことを示します。
  - *未配置* は、1 つ以上のインベントリーオブジェクトが配置されなかったことを示します。
- **アイテム数** は、ワークフロー、ジョブリソースなどのインベントリーオブジェクトの数を示します。

### インベントリーオブジェクトごとの配置履歴

インベントリーオブジェクトごとの詳細が表示されます：

- **メッセージ** は、配置に失敗した場合のエラーメッセージを示します。
- **オブジェクトタイプ** は、*ワークフロー*、*ジョブリソース* などのインベントリーオブジェクトのタイプを示します。
- **パス** はインベントリーフォルダとオブジェクト名を示します。ワークフローオブジェクトの場合
  - ワークフロー名をクリックすると[ワークフロー](/workflows) 画面に移動します、
  - 鉛筆アイコンをクリックすると[設定 - インベントリー - ワークフロー](/configuration-inventory-workflows) 画面に移動します。
- **操作** は、*保存* または *削除* のいずれかです。更新されたオブジェクトは両方の操作で発生します。
- **日付**は、デプロイ操作の時点です。
