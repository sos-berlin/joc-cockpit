# モニター - エージェントの可用性

このビューには、エージェントインスタンスの可用性のインジケータが表示されます。

Agentクラスタが使用されている場合、クラスタの可用性が考慮されます。例えば、クラスタ内のAgentインスタンスがメンテナンスのためにシャットダウンされ、残りのインスタンスが負荷を補う場合、可用性は低下しません。

画面右上には、[*Current Controller*] チェックボックスがあります。チェックを外すと、接続されているすべてのコントローラのエージェントの可用性が表示され、チェックを外すと、現在選択されているコントローラのみに登録されているエージェントの情報が表示されます。

エージェントの可用性の履歴データは、[Cleanup Service](/service-cleanup) によってパージされる可能性があることに注意してください。

## 日付フィルタ

パネルの右上隅では、空席状況を表示する日付範囲を選択することができます：

- **Week**は日付スライダーを1週間単位に切り替えます。
- **月**は1ヶ月単位でスライダーを切り替えます。
- **範囲**は開始日と終了日を指定します。

## 実行時間

指定された期間にエージェントが利用可能であると確認された割合を示します。

## 統計

指定された期間内の稼働状況を日毎に棒グラフで表示します。各エージェントは1日ごとに表示されます。 

## 概要

指定された期間のエージェントごと、日ごとの稼働率を表示します。

- チャートは、エージェントの可用性が確認された時間帯を緑色で表示します。 
- 赤色は利用不可を示します。
- 灰色はデータの欠落を示します。
